## 2. 产品概述

### 2.1 产品背景

随着教育信息化的深入推进，学校对学生阅读能力培养的重视程度不断提升。传统的阅读教学存在以下问题：

- 老师难以追踪每位学生的阅读进度和效果
- 缺乏有效的阅读激励机制
- 阅读数据分散，无法形成系统性分析

**现状痛点：**

- 老师无法实时了解学生阅读进度，只能依赖定期检查
- 阅读任务布置和跟踪依赖纸质材料，效率低下
- 缺乏科学的阅读测评体系，难以量化阅读效果
- 班级阅读数据分散，无法进行横向对比分析

**影响范围：**

- 影响全校语文老师的阅读教学工作
- 影响学生阅读习惯的养成和阅读能力的提升

"校校读吧"平台应运而生，旨在通过数字化手段提升学校阅读教学效率

**阅读计划来源：**

阅读计划由运营方在管理后台统一创建和发布，老师端不支持自定义创建阅读计划。运营方根据学期安排（如寒暑假、学期季度）发布阅读计划，每个计划按年级配置不同的书目（如3年级读3本书、4年级读4本书），每本书籍设置N次阅读（对应N天打卡），每次阅读包含观看视频、阅读指定页数、完成导读单等任务。老师通过报名功能为班级参与阅读计划，学生根据所在年级看到对应的书目，在学生端完成阅读任务并打卡

本文档聚焦于老师端产品的功能需求

### 2.2 产品定位

**校校读吧老师端**是一款面向中小学教师的阅读教学管理工具，通过数字化手段帮助老师高效组织班级阅读活动、实时监控学生阅读进度、科学评估阅读效果，提升阅读教学质量

### 2.3 产品目标

| 目标类型 | 具体目标                               | 衡量指标                                   |
| -------- | -------------------------------------- | ------------------------------------------ |
| 业务目标 | 支撑老师完成班级报名和学生阅读进度管理 | 老师报名流程完成率、班级阅读进度查看覆盖率 |
| 用户目标 | 让老师便捷查看学生阅读情况和测评成绩   | 核心操作路径步数、老师端用户满意度评分     |
| 技术目标 | 提供稳定流畅的微信小程序体验           | 首屏加载时间≤2秒、接口成功率≥99.9%       |

> 注：运营指标（如活跃率、报名率等）的统计需要管理后台支持，相关需求见管理后台PRD

### 2.4 目标用户

| 用户角色 | 特征描述                       | 核心诉求             | 使用场景           |
| -------- | ------------------------------ | -------------------- | ------------------ |
| 语文老师 | 负责班级阅读教学的一线老师     | 高效管理学生阅读任务 | 课间、办公室、家中 |
| 阅读老师 | 学校指定负责阅读推广的专职老师 | 组织班级阅读活动     | 阅读课、课外活动   |

### 2.5 项目范围

**本期包含：**

- 老师登录与身份认证
- 班级创建与学生管理
- 阅读计划报名
- 学生阅读进度监控
- 测评管理与成绩查看
- 数据报告查看
- 消息中心（接收系统通知）
- 个人中心设置

**本期不包含：**

- 自定义阅读计划创建 - 原因：后续版本迭代
- 积分商城兑换 - 原因：运营功能后续开发

> 注：积分功能本期仅包含积分查看（F026积分明细），不包含积分兑换功能。老师可查看积分获取记录，但暂不支持积分消费

**平台架构说明：**

"校校读吧"平台由三个端组成，各端职责如下：

| 端       | 用户      | 主要职责                                                               |
| -------- | --------- | ---------------------------------------------------------------------- |
| 管理后台 | 运营人员  | 创建和发布阅读计划、配置书籍资源、管理学校和老师、数据统计分析         |
| 老师端   | 学校老师  | 创建班级、为班级报名阅读计划、监控学生阅读进度、查看测评成绩和数据报告 |
| 学生端   | 学生/家长 | 完成阅读任务、观看视频、完成导读单、参与测评、打卡记录                 |

> 注：本文档为老师端PRD，学生端和管理后台将分别编写独立的需求文档

## 3. 用户分析

### 3.1 用户画像

**典型用户1：王老师（语文老师）**

- 基本信息：35岁，小学语文老师，教龄10年
- 行为特征：习惯使用手机处理工作事务，每天查看学生作业情况
- 痛点：管理3个班级150多名学生的阅读任务，工作量大
- 期望：能够快速了解学生阅读进度，及时发现落后学生并提醒

**典型用户2：陈老师（阅读老师）**

- 基本信息：32岁，小学阅读推广老师，教龄8年
- 行为特征：专注阅读教学，负责多个班级的阅读活动组织
- 痛点：同时管理多个班级的阅读进度，信息分散难以统一管理
- 期望：能够集中查看所有班级的阅读情况，高效组织阅读活动

> 注：当前阶段主要面向小学1-6年级，后续版本将扩展至初中、高中

### 3.2 用户场景

**场景1：课间查看学生阅读进度**

- 时间：课间休息时间（10分钟）
- 地点：办公室或教室
- 行为：打开小程序，快速查看班级学生今日打卡情况，发现未打卡学生并进行提醒
- 期望：操作简单快捷，一目了然看到学生完成情况

**场景2：周末查看班级阅读数据**

- 时间：周末在家
- 地点：家中
- 行为：查看本周班级阅读数据报告，了解整体进度和学生表现，为下周教学做准备
- 期望：数据清晰直观，能够发现问题学生，提供改进建议

**场景3：学期初为班级报名阅读计划**

- 时间：新学期开学前或开学初
- 地点：办公室
- 行为：浏览可报名的阅读计划，选择适合班级的计划进行报名
- 期望：计划信息详细，报名流程简单，支持批量为多个班级报名

**场景4：测评后查看学生成绩**

- 时间：测评结束后
- 地点：办公室或家中
- 行为：查看学生测评成绩，了解学生对书籍内容的理解程度，发现薄弱环节
- 期望：成绩展示清晰，提供AI智能分析，帮助了解学生能力水平

### 3.3 用户旅程

**场景：老师为班级报名阅读计划并监控学生进度**

| 阶段               | 登录系统   | 查看阅读计划 | 为班级报名     | 监控学生进度   | 查看测评成绩 | 查看数据报告 |
| ------------------ | ---------- | ------------ | -------------- | -------------- | ------------ | ------------ |
| **用户情绪** | 期待       | 好奇         | 满意           | 关注           | 欣慰/担忧    | 成就感       |
| **痛点1**    | 登录繁琐   | 计划太多     | 班级多选麻烦   | 数据更新不及时 | 成绩解读困难 | 数据维度单一 |
| **痛点2**    | 记不住密码 | 不知如何选择 | 不确定是否成功 | 找不到落后学生 | 缺乏改进建议 | 数据不够详细 |

**用户旅程流程图：**

```
[登录系统] ──▶ [查看阅读计划] ──▶ [为班级报名] ──▶ [监控学生进度] ──▶ [查看测评成绩] ──▶ [查看数据报告]
```

**关键触点优化：**

| 阶段         | 用户期望         | 设计要点                       |
| ------------ | ---------------- | ------------------------------ |
| 登录系统     | 快速进入         | 支持微信一键登录，记住登录状态 |
| 查看阅读计划 | 快速找到合适计划 | 清晰的状态标签，支持搜索筛选   |
| 为班级报名   | 操作简单明确     | 支持批量选择，实时反馈报名结果 |
| 监控学生进度 | 一目了然         | 可视化进度条，支持排序筛选     |
| 查看测评成绩 | 了解学生表现     | AI智能评价，多维度能力分析     |
| 查看数据报告 | 了解班级整体情况 | 数据可视化，多维度分析         |

### 4.5 业务模型

#### 4.5.1 阅读计划结构

阅读计划采用层级结构组织，具体如下：

```
阅读计划
├── 3年级书目
│   ├── 书籍1
│   │   ├── 第1次阅读（Day 1）
│   │   │   ├── 任务1：观看导读视频
│   │   │   ├── 任务2：阅读指定页数
│   │   │   └── 任务3：完成导读单
│   │   ├── 第2次阅读（Day 2）
│   │   │   ├── 任务1：观看分享视频
│   │   │   ├── 任务2：阅读指定页数
│   │   │   └── 任务3：完成导读单
│   │   ├── ...
│   │   └── 第N次阅读（Day N）- 最后一次
│   │       ├── 任务1：观看分享视频
│   │       ├── 任务2：阅读指定页数
│   │       ├── 任务3：完成导读单
│   │       └── 任务4：完成测评题
│   ├── 书籍2
│   │   └── ...
│   └── 书籍N
│       └── ...
├── 4年级书目
│   ├── 书籍1
│   │   └── ...
│   └── 书籍N
│       └── ...
└── ...其他年级
```

**层级说明：**

| 层级     | 说明                                   | 数量               |
| -------- | -------------------------------------- | ------------------ |
| 阅读计划 | 按学期或假期发布，如"2026寒假阅读计划" | 运营配置           |
| 年级书目 | 每个年级配置不同的书籍列表             | 运营配置（按年级） |
| 书籍     | 该年级下的书籍列表                     | 运营配置（不固定） |
| 阅读次数 | 每本书的阅读次数，对应学生打卡次数     | 运营配置           |
| 任务     | 每次阅读包含的具体任务项               | 运营配置           |

#### 4.5.2 打卡机制

- 学生完成当前阅读计划下当次阅读中的导读单任务后进行打卡
- 同一阅读计划下的同一次阅读所对应的导读单任务仅允许提交一次打卡
- 同一天若存在多个阅读计划且各自包含导读单任务，学生需分别完成打卡
- 打卡记录用于统计学生阅读进度和连续打卡天数

#### 4.5.3 测评机制

- 每本书籍配置一套测评题
- 测评题在该书籍最后一次阅读时开放
- 最后一次阅读除测评外，可能还包含其他任务（视频、阅读、导读单等）
- 测评成绩用于评估学生对书籍内容的理解程度

#### 4.5.4 资源与测评开放机制

阅读资源（视频、导读单）和测评题的开放状态由管理后台配置：

| 开放策略    | 说明                                |
| ----------- | ----------------------------------- |
| 当天开放    | 资源/测评在对应阅读日当天开放       |
| 提前N天开放 | 资源/测评在对应阅读日前N天开放      |
| 锁定        | 资源/测评未到开放时间，显示锁定状态 |

> 注：老师端展示资源和测评的开放/锁定状态，具体开放规则在管理后台配置

**功能说明：**
监控班级学生的阅读进度和打卡情况，支持多维度筛选。页面包含两个页签：

- **今日打卡情况**：查看所选班级在指定日期、指定阅读计划下的学生打卡完成情况，支持切换已完成/未完成学生列表，可对未完成学生发送提醒
- **计划完成情况**：查看整体阅读进度，对比实际进度与预期进度，支持切换全部/正常/延期学生列表

**数据来源：**
进度数据从管理后台获取，实时同步学生打卡状态

**业务规则：**

1. 今日打卡情况页签的筛选条件：班级、日期、阅读计划
2. 计划完成情况页签的筛选条件：阅读计划、班级、当前书籍
3. 班级选项根据所选计划动态更新（只显示已报名该计划的班级）
4. 书籍选项根据所选计划动态更新
5. 今日打卡情况页签默认显示"已完成"学生列表，支持切换到"未完成"
6. 计划完成情况页签默认显示"全部"学生列表，支持切换到"正常"或"延期"
7. 阅读计划选项来源于所选班级已报名且在所选日期有效的阅读计划；若仅有一个阅读计划，则默认选中且无需额外筛选
8. 今日任务内容根据所选阅读计划在所选日期开放的阅读关卡自动生成，可能包含阅读、观看视频、完成导读单、测评等任务类型
9. 今日打卡情况页签的已完成/未完成判断基于当前筛选的阅读计划维度：学生完成该阅读计划在所选日期下需完成的打卡任务后记为"已完成"，否则记为"未完成"
10. 打卡动态入口点击后跳转到F012 学生打卡动态页面（39-学生打卡动态.html），展示学生的打卡分享、点赞、评论等互动信息
11. 完成统计数据实时计算，包含应完成人数（班级总人数）、已完成人数、未完成人数
12. 整体进度对比计算规则：
    - 已完成阅读次数判定：学生完成该次阅读下所有已开放的任务（任务类型因关卡配置而异，可能包含观看视频、阅读、完成导读单中的一种或多种），即视为完成该次阅读。测评属于整本书维度，不纳入单次阅读的完成判定
    - 实际进度 = 班级所有学生已完成的阅读次数之和 ÷（班级学生数 × 该书籍总阅读次数）× 100%
    - 应达进度 = 截至当前日期已开放的阅读次数 ÷ 该书籍总阅读次数 × 100%
    - 示例：某书籍共8次阅读，班级13人，当前已开放6次，全班累计完成69次，则实际进度 = 69 ÷（13×8）= 66%，应达进度 = 6÷8 = 75%
13. 学生完成详情支持折叠/展开，默认展开状态

15. 提醒弹窗展示固定的提醒文案模板，不可编辑
16. 点击"立即发送"按钮发送提醒，发送成功后Toast提示"发送成功"
17. 进度数据中的年级信息使用报名时记录的快照年级，不随班级年级自动升级而变化

**埋点目的：**

**埋点分类：**

| 埋点类型   | 说明                             | 示例                              |
| ---------- | -------------------------------- | --------------------------------- |

- 每个事件包含公共字段：teacher_id、timestamp、device_info、app_version、network_type

### 10.3 通用交互规范

**导航栏：**

- 高度以UI图为准
- 左侧返回按钮，中间标题，右侧操作按钮（可选）

**列表交互：**

- 下拉刷新：触发数据重新加载
- 上拉加载：分页加载更多数据
- 左滑操作：显示删除等操作按钮

**弹窗类型：**

- Modal弹窗：居中显示，半透明遮罩
- 底部弹窗：从底部滑出，圆角顶部
- Toast提示：居中短暂显示，2秒后消失

**按钮状态：**

- 默认态：正常显示
- 点击态：透明度降低或缩放效果
- 禁用态：不可点击

### 10.4 弹窗组件规范

#### 10.4.1 弹窗分类

| 类型      | 说明                                     | 样式特征                        |
| --------- | ---------------------------------------- | ------------------------------- |
| Modal弹窗 | 居中显示的对话框，用于确认操作、信息展示 | 居中显示，半透明遮罩，圆角卡片  |
| 底部弹窗  | 从底部滑出的面板，用于选择、筛选等操作   | 底部滑出，圆角顶部，可下滑关闭  |
| 全屏弹窗  | 覆盖整个屏幕，用于预览、播放等沉浸式场景 | 全屏覆盖，深色背景，手势关闭    |
| Toast提示 | 轻量级提示，短暂显示后自动消失           | 居中显示，2秒后消失，不阻断操作 |

#### 10.4.2 功能流程弹窗

##### P01 快捷创建班级弹窗

**触发场景：** 老师点击阅读计划的"报名"按钮时，若尚未创建任何班级

**触发条件：**

- 老师点击计划报名按钮（阅读计划列表页或计划详情页）
- 系统检测到该老师名下没有班级记录
- 只要没有班级，每次点击报名都会触发，不仅限于首次登录

**弹窗类型：** Modal弹窗

**界面要素：**

| 元素     | 类型     | 必填 | 说明                                   |
| -------- | -------- | ---- | -------------------------------------- |
| 弹窗标题 | 文本     | -    | "快捷创建班级"                         |
| 学校选择 | 选择器   | 是   | 若已有学校信息则直接显示，否则需要选择 |
| 年级选择 | 下拉选择 | 是   | 一年级至六年级，共6个选项              |
| 班级选择 | 下拉选择 | 是   | 1-99班或自定义输入                     |
| 学生人数 | 输入框   | 否   | 可选填写班级学生人数                   |
| 取消按钮 | 按钮     | -    | 关闭弹窗                               |
| 确定按钮 | 按钮     | -    | 创建班级并完成报名                     |

**交互规则：**

1. 若老师已在个人中心设置过学校，弹窗中直接显示学校名称，无需重新选择
2. 年级和班级均选择后，确定按钮可点击
3. 班级名称自动生成，格式为"X年级(X)班"
4. 支持选择"自定义"后输入班级名称，如"三年级实验班"
5. 点击确定后：创建班级 + 自动完成报名（一步到位）
6. 创建成功后弹窗关闭，显示"报名成功"Toast提示

**异常处理：**

| 异常场景     | 处理方式   | 提示文案         |
| ------------ | ---------- | ---------------- |
| 班级名称重复 | 弹窗内提示 | 该班级已存在     |
| 创建失败     | Toast提示  | 创建失败，请重试 |
| 报名失败     | Toast提示  | 报名失败，请重试 |
