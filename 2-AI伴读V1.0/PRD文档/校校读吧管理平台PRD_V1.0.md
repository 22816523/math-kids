## 2. 产品概述

### 2.1 产品背景

随着"校校读吧"平台的推广，需要一个统一的管理后台来支撑平台的运营工作

**核心痛点：**

- 阅读计划创建和发布流程繁琐，缺乏统一管理
- 书籍资源分散，难以形成系统化的资源库
- 学校和老师数据管理缺乏规范，无法有效监控
- 学生阅读数据分散，无法进行深度分析

"校校读吧管理平台"旨在为平台管理人员提供一站式的运营管理工具，提升平台运营效率和服务质量

### 2.2 产品定位

**校校读吧管理平台**是一款面向平台管理人员的运营管理工具，通过集中化的管理界面，支持阅读计划创建与发布、书籍资源库管理、学校与老师管理、学生数据管理、数据统计分析等核心功能，帮助管理人员高效完成平台运营工作

### 2.3 产品目标

| 目标类型 | 具体目标                                       |
| -------- | ---------------------------------------------- |
| 业务目标 | 支撑平台管理人员完成阅读计划创建和资源管理     |
| 用户目标 | 让管理人员便捷管理学校、老师、学生和阅读数据   |
| 技术目标 | 提供稳定高效的Web管理后台，支持大数据量处理   |

### 2.4 目标用户

| 用户角色     | 特征描述                       | 核心诉求                 | 使用场景       |
| ------------ | ------------------------------ | ------------------------ | -------------- |
| 平台管理人员 | 负责平台日常运营和内容管理     | 高效创建和发布阅读计划   | 办公室、远程   |
| 平台管理人员 | 负责书籍资源审核和上架         | 规范管理书籍资源库       | 办公室         |
| 平台管理人员 | 负责学校和老师的管理与服务     | 监控学校参与和老师活跃度 | 办公室、远程   |
| 平台管理人员 | 负责数据分析和运营决策支持     | 获取准确及时的数据报表   | 办公室、会议室 |

### 2.5 项目范围

**本期包含：**

- 登录与权限管理（F001-F002：账号密码登录、企微单点登录）
- 首页（F003-F005：数据概览、快捷入口、最近动态）
- 用户管理（F006-F013：学校管理、班级管理、小程序用户管理）
- 阅读计划管理（F014-F018：计划列表、计划详情、配置年级书目、编辑计划、发布与撤回）
- 书籍资源库（F019-F026：书籍列表、详情、新增/编辑、资源管理、测评配置、AI虚拟形象、上下架）
- 激励体系（F027-F029：积分规则、等级体系、证书配置）
- 消息管理（F030：消息列表）
- 用户反馈（F031-F033：反馈列表、详情、回复）
- 数据分析（F034-F037：平台/学校/年级/学生数据分析）
- 系统配置（F038-F039：系统参数、数据字典）

**本期不包含：**

- 效率工具（批量导入导出）- 集成在各功能界面中，不作为独立模块
- 操作日志查询 - 后续版本迭代

**说明：**

- 权限管理由云平台统一配置，管理平台不单独管理权限
- 阅读计划支持按年级配置不同书目（计划→年级→书籍层级）
- 阅读计划到期后自动结束

**平台架构说明：**

"校校读吧"平台由三个端组成，各端职责如下：

| 端       | 用户             | 主要职责                                                               |
| -------- | ---------------- | ---------------------------------------------------------------------- |
| 管理平台 | 平台管理人员     | 创建和发布阅读计划、配置书籍资源、管理学校和老师、数据统计分析         |
| 老师端   | 学校老师         | 创建班级、为班级报名阅读计划、监控学生阅读进度、查看测评成绩和数据报告 |
| 学生端   | 学生/家长        | 完成每日阅读任务、观看视频、完成导读单、参与测评、打卡记录             |

> 注：本文档为管理平台PRD，老师端和学生端已有独立的需求文档

## 3. 用户分析

### 3.1 用户画像

**典型用户1：张经理（平台运营负责人）**

- 基本信息：38岁，负责平台整体运营，工作经验15年
- 行为特征：习惯使用电脑办公，每天查看平台运营数据
- 痛点：需要统筹多个阅读计划的创建和发布，工作量大
- 期望：能够快速创建阅读计划，灵活配置书籍和资源，实时查看运营数据

**典型用户2：李老师（内容运营专员）**

- 基本信息：28岁，负责书籍资源管理和测评配置，工作经验5年
- 行为特征：每天上传和审核书籍资源，配置测评题目
- 痛点：书籍资源上传流程繁琐，测评题目录入效率低
- 期望：能够批量上传资源，快速配置测评，支持模板导入

**典型用户3：王主管（客户服务负责人）**

- 基本信息：35岁，负责学校和老师的管理与服务，工作经验10年
- 行为特征：每天查看学校参与情况和老师活跃度，处理老师反馈
- 痛点：学校和老师数据分散，难以快速定位问题
- 期望：能够集中查看学校和老师信息，快速响应问题

### 3.2 用户场景

**场景1：创建新学期阅读计划**

- 时间：新学期开始前1-2周
- 地点：办公室
- 行为：登录管理平台，创建新的阅读计划，选择适合的书籍，配置每本书的阅读天数和任务，设置资源解锁时间，配置AI导语，发布计划
- 期望：操作流程清晰，支持复制历史计划，快速完成配置

**场景2：上传新书籍资源**

- 时间：日常工作时间
- 地点：办公室
- 行为：登录管理平台，上传书籍封面和基本信息，上传配套的视频、音频、导读单等资源，配置测评题目，提交审核
- 期望：支持批量上传，自动校验资源格式，提供上传进度反馈

**场景3：查看学校参与情况**

- 时间：每周例会前
- 地点：办公室
- 行为：登录管理平台，查看学校列表，筛选参与阅读计划的学校，查看各学校的老师数量和学生数量，导出数据报表
- 期望：数据实时更新，支持多维度筛选，快速导出报表

**场景4：监控阅读计划进展**

- 时间：阅读计划进行中
- 地点：办公室或远程
- 行为：登录管理平台，查看阅读计划的报名情况，查看各学校的参与进度，查看学生打卡率和测评完成率
- 期望：数据可视化展示，支持下钻查看详情，及时发现异常

### 3.3 用户旅程

**场景：平台管理人员创建并发布阅读计划**

| 阶段             | 登录系统   | 创建计划   | 配置书籍   | 设置资源   | 发布计划   | 监控进展   |
| ---------------- | ---------- | ---------- | ---------- | ---------- | ---------- | ---------- |
| **用户情绪**     | 期待       | 专注       | 细致       | 谨慎       | 满意       | 关注       |
| **痛点1**        | 登录繁琐   | 不知如何开始 | 书籍选择困难 | 资源配置复杂 | 不确定是否成功 | 数据更新不及时 |
| **痛点2**        | 权限不清晰 | 缺乏模板参考 | 资源不全   | 解锁策略难理解 | 缺乏发布反馈 | 数据维度单一 |

**用户旅程流程图：**

```
[登录系统] ──▶ [创建计划] ──▶ [配置书籍] ──▶ [设置资源] ──▶ [发布计划] ──▶ [监控进展]
```

**关键触点优化：**

| 阶段     | 用户期望         | 设计要点                       |
| -------- | ---------------- | ------------------------------ |
| 登录系统 | 快速进入         | 支持企微单点登录，记住登录状态 |
| 创建计划 | 快速上手         | 提供创建向导，支持复制历史计划 |
| 配置书籍 | 快速找到合适书籍 | 清晰的分类标签，支持搜索筛选   |
| 设置资源 | 灵活配置         | 可视化配置界面，提供预设模板   |
| 发布计划 | 操作简单明确     | 一键发布，实时反馈发布结果     |
| 监控进展 | 一目了然         | 数据可视化，支持多维度分析     |

### 4.4 业务模型

#### 4.4.1 阅读计划结构

阅读计划是平台的核心业务对象，其结构如下：

```
阅读计划
├── 计划基本信息
│   ├── 计划名称
│   ├── 适用年级（1-6年级）
│   ├── 开始时间
│   ├── 结束时间
│   └── 计划状态（草稿/已发布/已结束/已下架）
├── 书籍列表（1-N本书）
│   └── 书籍
│       ├── 书籍基本信息（从资源库选择）
│       ├── 阅读天数（N天）
│       ├── AI导语
│       └── 每日任务列表（1-N天）
│           └── 每日任务
│               ├── 视频资源（可选）
│               ├── 音频资源（可选）
│               ├── 导读单资源（可选）
│               ├── 阅读页数（必填）
│               └── 资源解锁策略
│                   ├── 解锁时间类型（立即解锁/定时解锁/完成前置任务后解锁）
│                   └── 解锁时间参数
└── 统计数据
    ├── 报名班级数
    ├── 参与学校数
    ├── 参与学生数
    └── 整体完成率
```

#### 4.4.2 资源解锁机制

资源解锁策略支持以下三种类型：

1. **立即解锁**：学生报名后立即可以访问该资源
2. **定时解锁**：在指定的日期时间后解锁（如：2026-02-15 08:00）
3. **完成前置任务后解锁**：完成指定的前置任务后解锁（如：完成第1天打卡后解锁第2天视频）

#### 4.4.3 书籍资源结构

书籍资源库中的书籍结构如下：

```
书籍
├── 基本信息
│   ├── 书名
│   ├── 作者
│   ├── 出版社
│   ├── ISBN
│   ├── 适用年级
│   ├── 书籍分类
│   └── 书籍封面
├── 配套资源
│   ├── 视频资源列表
│   ├── 音频资源列表
│   └── 导读单资源列表
├── 测评配置
│   └── 测评题目列表
│       └── 题目
│           ├── 题目类型（单选/多选/判断/填空）
│           ├── 题目内容
│           ├── 选项列表（选择题）
│           ├── 正确答案
│           └── 题目解析
└── 状态信息
    ├── 上架状态
    └── 创建时间
```

### 5.1 登录与权限管理

#### 5.1.1 F001 账号密码登录

**功能描述**

支持平台管理人员使用账号和密码登录管理平台

**业务规则**

1. 账号格式：支持手机号或邮箱作为账号
2. 密码规则：
   - 长度：8-20位
   - 复杂度：必须包含字母、数字、特殊字符中的至少两种
3. 登录失败处理：
   - 连续输入错误密码5次，账号锁定30分钟
   - 锁定期间显示剩余解锁时间
4. 登录成功后：
   - 生成登录token，有效期24小时
   - 记录登录日志（登录时间、IP地址、设备信息）
   - 跳转到首页（默认为阅读计划列表页）

**页面元素**

| 元素名称     | 元素类型 | 必填 | 说明                           |
| ------------ | -------- | ---- | ------------------------------ |
| 账号输入框   | 文本框   | 是   | 提示文字：请输入手机号或邮箱   |
| 密码输入框   | 密码框   | 是   | 提示文字：请输入密码           |
| 记住密码     | 复选框   | 否   | 勾选后7天内自动登录            |
| 忘记密码链接 | 链接     | 否   | 点击跳转到密码重置页面         |
| 登录按钮     | 按钮     | -    | 点击提交登录请求               |
| 企微登录按钮 | 按钮     | -    | 点击跳转到企业微信登录授权页面 |

**交互说明**

1. 输入账号和密码后，点击"登录"按钮
2. 系统验证账号和密码是否正确
3. 验证通过后，跳转到首页
4. 验证失败，显示错误提示：
   - 账号不存在：提示"账号不存在，请检查后重试"
   - 密码错误：提示"密码错误，还可尝试X次"
   - 账号被锁定：提示"账号已被锁定，请X分钟后重试"
   - 账号被禁用：提示"账号已被禁用，请联系管理员"

**异常处理**

| 异常场景         | 处理方式                                   |
| ---------------- | ------------------------------------------ |
| 网络异常         | 提示"网络连接失败，请检查网络后重试"       |
| 服务器异常       | 提示"服务器异常，请稍后重试"               |
| 账号不存在       | 提示"账号不存在，请检查后重试"             |
| 密码错误         | 提示"密码错误，还可尝试X次"                |
| 账号被锁定       | 提示"账号已被锁定，请X分钟后重试"          |
| 账号被禁用       | 提示"账号已被禁用，请联系管理员"           |

3. 数据预填充：
   - 步骤1：年级下拉框默认选中已配置的年级，已选书籍和开始时间预填充
   - 步骤2：阅读次数、任务配置、资源文件、解锁策略全部预填充
   - 步骤3：预览展示当前最新配置
4. 编辑权限：
   - 未发布（草稿）：所有内容均可编辑
   - 已发布/待开始：所有内容均可编辑，保存后需重新发布
   - 进行中：仅可编辑未开始的书籍和任务，已开始的部分不可修改
   - 已结束：不可编辑，入口隐藏
5. 操作按钮：

#### 5.4.5 F018 计划发布与撤回

**功能描述**

发布阅读计划供老师报名，或撤回已发布的计划

**业务规则**

1. 发布条件：
   - 计划状态为"未发布"（草稿）
   - 至少配置了一个年级的书目
   - 每个已配置年级至少有一本书籍
   - 每本书至少配置了一次阅读任务
2. 发布流程：
   - 点击"发布"按钮，弹出确认弹窗
   - 确认弹窗提示"确认要发布此阅读计划吗？阅读计划发布后会显示在小程序界面上，老师能够看到并报名参与"
   - 确认后发布成功，弹出成功提示
   - 成功提示提供"返回列表"和"查看详情"两个入口
3. 发布后：
   - 计划状态变为"已发布"
   - 老师端可见该计划并可报名
4. 撤回规则：
   - 已发布但未开始的计划可以撤回
   - 撤回后计划状态变为"未发布"
   - 已报名的班级将收到撤回通知
   - 进行中的计划不可撤回

**页面元素**

| 元素名称         | 元素类型 | 说明                           |

| 发布成功弹窗     | 弹窗     | 展示发布成功信息               |
| 撤回确认弹窗     | 弹窗     | 确认撤回操作                   |

**交互说明**

1. 在计划列表页，未发布的计划显示"发布"按钮
2. 点击"发布"弹出确认弹窗
3. 点击"确认发布"执行发布
4. 发布成功后弹出成功提示，可选择"返回列表"或"查看详情"
5. 已发布的计划显示"撤回"按钮
6. 点击"撤回"弹出确认弹窗，确认后执行撤回

3. 支持按消息类型、发送时间筛选

**交互说明**

1. 进入消息列表页面，查看所有消息记录
2. 点击消息标题查看详情
3. 支持筛选和搜索

### 5.8 用户反馈

#### 5.8.1 F031 反馈列表

**功能描述**

查看用户提交的意见反馈，支持筛选和处理

**业务规则**

1. 列表展示：
   - 默认按提交时间倒序排列
   - 每页支持10/20/50/100条记录切换
2. 筛选条件：
   - 反馈类型：全部/功能建议/问题反馈/其他
   - 处理状态：全部/待处理/处理中/已处理
   - 提交时间：日期范围
3. 反馈状态流转：待处理 → 处理中 → 已处理

**列表字段**

| 字段名称         | 字段说明                       |
| ---------------- | ------------------------------ |
| 反馈ID           | 系统生成的唯一ID               |
| 反馈类型         | 功能建议/问题反馈/其他         |
| 反馈内容         | 反馈内容摘要                   |
| 提交人           | 提交反馈的用户                 |
| 提交时间         | 反馈提交时间                   |
| 处理状态         | 待处理/处理中/已处理           |
| 操作             | 查看/回复                      |

**交互说明**

1. 进入反馈列表页面，默认显示所有反馈
2. 使用筛选条件缩小范围
3. 点击"查看"进入反馈详情
4. 点击"回复"进入回复页面

#### 5.8.2 F032 反馈详情

**功能描述**

查看反馈的完整内容和处理记录

**业务规则**

1. 展示反馈完整内容、附件图片
2. 展示处理记录和回复历史
3. 可修改处理状态

**交互说明**

1. 从反馈列表点击"查看"进入
2. 查看反馈详情和处理记录
3. 点击"返回列表"回到反馈列表

#### 5.8.3 F033 反馈回复

**功能描述**

回复用户反馈并更新处理状态

**业务规则**

1. 回复内容不超过500个字符
2. 回复后自动将状态更新为"已处理"
3. 回复内容推送给反馈提交人

**交互说明**

1. 从反馈列表或详情页点击"回复"
2. 输入回复内容
3. 点击"提交"发送回复

### 5.10 系统配置

#### 5.10.1 F038 系统参数配置

**功能描述**

配置系统运行的基础参数

**业务规则**

1. 参数分类：
   - 基础参数：平台名称、Logo、联系方式等
   - 业务参数：默认积分值、打卡时间范围、文件大小限制、每个班级学生人数上限等

| 元素名称         | 元素类型 | 说明                           |

| 保存按钮         | 按钮     | 保存参数配置                   |

**交互说明**

2. 切换标签查看不同分类
3. 修改参数值后点击"保存"

#### 5.10.2 F039 数据字典

**功能描述**

管理系统中使用的数据字典项，如年级列表、标签分类等

**业务规则**

1. 字典分类：
   - 年级列表：一年级~六年级
   - 书籍标签：文学/科幻/历史/科普等
   - 阅读能力标签：信息提取/理解分析/推理判断等
   - 反馈类型：功能建议/问题反馈/其他
2. 支持新增、编辑、删除字典项
3. 已被引用的字典项不可删除

**页面元素**

| 元素名称         | 元素类型 | 说明                           |
| ---------------- | -------- | ------------------------------ |
| 字典分类列表     | 列表     | 展示字典分类                   |
| 字典项列表       | 表格     | 展示选中分类的字典项           |
| 新增字典项按钮   | 按钮     | 添加新字典项                   |

**交互说明**

1. 进入数据字典页面，左侧显示分类列表
2. 点击分类，右侧显示该分类下的字典项
3. 可新增、编辑、删除字典项

**异常处理**

| 异常场景             | 处理方式                                   |
| -------------------- | ------------------------------------------ |
| 删除已引用的字典项   | 提示"该字典项已被引用，不可删除"           |
| 保存失败             | 提示"保存失败，请重试"                     |
