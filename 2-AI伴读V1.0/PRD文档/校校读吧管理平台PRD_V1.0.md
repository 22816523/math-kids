## 2. 产品概述

### 2.1 产品背景

随着"校校读吧"平台的推广，需要一个统一的管理后台来支撑平台的运营工作

**核心痛点：**

- 阅读计划创建和发布流程繁琐，缺乏统一管理
- 书籍资源分散，难以形成系统化的资源库
- 学校和老师数据管理缺乏规范，无法有效监控
- 学生阅读数据分散，无法进行深度分析

"校校读吧管理平台"旨在为平台管理人员提供一站式的运营管理工具，提升平台运营效率和服务质量

### 2.2 产品定位

**校校读吧管理平台**是一款面向平台管理人员的运营管理工具，通过集中化的管理界面，支持阅读计划创建与发布、书籍资源库管理、学校与老师管理、学生数据管理、数据统计分析等核心功能，帮助管理人员高效完成平台运营工作

### 2.3 产品目标

| 目标类型 | 具体目标                                     | 衡量指标                                         |
| -------- | -------------------------------------------- | ------------------------------------------------ |
| 业务目标 | 支撑平台管理人员完成阅读计划创建和资源管理   | 阅读计划创建效率≤30分钟/个，书籍录入≤10分钟/本 |
| 用户目标 | 让管理人员便捷管理学校、老师、学生和阅读数据 | 核心操作路径≤3步，用户满意度≥4.0/5.0           |
| 技术目标 | 提供稳定高效的Web管理后台，支持大数据量处理  | 页面加载≤2秒，接口响应≤500ms，可用性≥99.5%    |

### 2.4 目标用户

| 用户角色     | 特征描述                   | 核心诉求                 | 使用场景       |
| ------------ | -------------------------- | ------------------------ | -------------- |
| 平台管理人员 | 负责平台日常运营和内容管理 | 高效创建和发布阅读计划   | 办公室、远程   |
| 平台管理人员 | 负责书籍资源审核和管理     | 规范管理书籍资源库       | 办公室         |
| 平台管理人员 | 负责学校和老师的管理与服务 | 监控学校参与和老师活跃度 | 办公室、远程   |
| 平台管理人员 | 负责数据分析和运营决策支持 | 获取准确及时的数据报表   | 办公室、会议室 |

### 2.5 项目范围

**本期包含：**

- 登录与权限管理（F001-F002：账号密码登录、企微单点登录）
- 首页（F003-F005：数据概览、快捷入口、最近动态）
- 用户管理（F006-F013：学校管理、班级管理、小程序用户管理）
- 阅读计划管理（F014-F018：计划列表、计划详情、配置年级书目、编辑年级书目、发布与撤回）
- 书籍资源库（F019-F026：书籍列表、详情、新增书籍、编辑书籍、上传资源、测评配置、AI虚拟形象）
- 激励体系（F027-F029：积分规则、等级体系、证书配置）
- 消息管理（F030：消息列表）
- 用户反馈（F031-F033：反馈列表、详情、回复）
- 数据分析（F034-F041：平台/学校/年级/学生数据分析、书籍测评记录、测评详情、阅读任务记录、导读单任务详情）
- 系统配置（F042-F044：系统参数、数据字典、AI导语配置）

**本期不包含：**

- 效率工具（批量导入导出）- 集成在各功能界面中，不作为独立模块
- 操作日志查询界面 - 后续版本迭代（后台仍需记录完整操作日志，前端本期暂不提供查询页面）

**说明：**

- 权限管理由云平台统一配置，管理平台不单独管理权限
- 阅读计划支持按年级配置不同书目（计划→年级→书籍层级）
- 阅读计划到期后自动结束

**平台架构说明：**

"校校读吧"平台由三个端组成，各端职责如下：

| 端       | 用户         | 主要职责                                                               |
| -------- | ------------ | ---------------------------------------------------------------------- |
| 管理平台 | 平台管理人员 | 创建和发布阅读计划、配置书籍资源、管理学校和老师、数据统计分析         |
| 老师端   | 学校老师     | 创建班级、为班级报名阅读计划、监控学生阅读进度、查看测评成绩和数据报告 |
| 学生端   | 学生/家长    | 查看阅读计划任务清单、完成阅读/视频/导读单/测评等任务，并在完成导读单任务后提交打卡记录 |

> 注：本文档为管理平台PRD，老师端和学生端已有独立的需求文档

## 3. 用户分析

### 3.1 用户画像

**典型用户1：张经理（平台运营负责人）**

- 基本信息：38岁，负责平台整体运营，工作经验15年
- 行为特征：习惯使用电脑办公，每天查看平台运营数据
- 痛点：需要统筹多个阅读计划的创建和发布，工作量大
- 期望：能够快速创建阅读计划，灵活配置书籍和资源，实时查看运营数据

**典型用户2：李老师（内容运营专员）**

- 基本信息：28岁，负责书籍资源管理和测评配置，工作经验5年
- 行为特征：每天上传和审核书籍资源，配置测评题目
- 痛点：书籍资源上传流程繁琐，测评题目录入效率低
- 期望：能够批量上传资源，快速配置测评，支持模板导入

**典型用户3：王主管（客户服务负责人）**

- 基本信息：35岁，负责学校和老师的管理与服务，工作经验10年
- 行为特征：每天查看学校参与情况和老师活跃度，处理老师反馈
- 痛点：学校和老师数据分散，难以快速定位问题
- 期望：能够集中查看学校和老师信息，快速响应问题

### 3.2 用户场景

**场景1：创建新学期阅读计划**

- 时间：新学期开始前1-2周
- 地点：办公室
- 行为：登录管理平台，创建新的阅读计划，选择适合的书籍，配置每本书的阅读天数和任务，设置资源解锁时间，配置AI导语，发布计划
- 期望：操作流程清晰，通过创建向导快速完成配置

**场景2：上传新书籍资源**

- 时间：日常工作时间
- 地点：办公室
- 行为：登录管理平台，上传书籍封面和基本信息，上传配套的视频、音频、导读单等资源，配置测评题目，提交审核
- 期望：支持批量上传，自动校验资源格式，提供上传进度反馈

**场景3：查看学校参与情况**

- 时间：每周例会前
- 地点：办公室
- 行为：登录管理平台，查看学校列表，筛选参与阅读计划的学校，查看各学校的老师数量和学生数量，导出数据报表
- 期望：数据实时更新，支持多维度筛选，快速导出报表

**场景4：监控阅读计划进展**

- 时间：阅读计划进行中
- 地点：办公室或远程
- 行为：登录管理平台，查看阅读计划的报名情况，查看各学校的参与进度，查看学生打卡率和测评完成率
- 期望：数据可视化展示，支持下钻查看详情，及时发现异常

### 3.3 用户旅程

**场景：平台管理人员创建并发布阅读计划**

| 阶段               | 登录系统   | 创建计划     | 配置书籍     | 设置资源       | 发布计划       | 监控进展       |
| ------------------ | ---------- | ------------ | ------------ | -------------- | -------------- | -------------- |
| **用户情绪** | 期待       | 专注         | 细致         | 谨慎           | 满意           | 关注           |
| **痛点1**    | 登录繁琐   | 不知如何开始 | 书籍选择困难 | 资源配置复杂   | 不确定是否成功 | 数据更新不及时 |
| **痛点2**    | 权限不清晰 | 缺乏模板参考 | 资源不全     | 解锁策略难理解 | 缺乏发布反馈   | 数据维度单一   |

**用户旅程流程图：**

```
[登录系统] ──▶ [创建计划] ──▶ [配置书籍] ──▶ [设置资源] ──▶ [发布计划] ──▶ [监控进展]
```

**关键触点优化：**

| 阶段     | 用户期望         | 设计要点                                                 |
| -------- | ---------------- | -------------------------------------------------------- |
| 登录系统 | 快速进入         | 支持从企微工作台免登录进入，保留账号密码登录作为补充入口 |
| 创建计划 | 快速上手         | 提供创建向导，帮助运营人员按步骤完成配置                 |
| 配置书籍 | 快速找到合适书籍 | 清晰的分类标签，支持搜索筛选                             |
| 设置资源 | 灵活配置         | 可视化配置界面，提供预设模板                             |
| 发布计划 | 操作简单明确     | 一键发布，实时反馈发布结果                               |
| 监控进展 | 一目了然         | 数据可视化，支持多维度分析                               |

### 4.4 业务模型

#### 4.4.1 阅读计划结构

阅读计划是平台的核心业务对象，其结构如下：

```
阅读计划
├── 计划基本信息
│   ├── 计划名称
│   ├── 计划封面
│   ├── 计划描述
│   ├── 开始时间
│   ├── 结束时间
│   ├── 计划状态（未发布/待开始/进行中/已结束）
│   └── 适用年级范围（一个计划可配置多个年级）
├── 年级配置（1-N个年级）
│   └── 年级
│       ├── 年级名称
│       ├── 年级书籍列表（1-N本书）
│       └── 年级统计信息
│           ├── 书籍数量
│           └── 任务数量
├── 书籍配置（按年级独立配置）
│   └── 书籍
│       ├── 书籍基本信息（从资源库选择）
│       ├── 阅读开始时间
│       ├── 阅读次数列表（1-N次）
│       │   └── 单次阅读
│       │       ├── 阅读标题（如第1次阅读）
│       │       ├── 系统任务日期（后台自动生成开始日期/结束日期）
│       │       ├── 任务列表（动态添加）
│       │       │   ├── 阅读任务
│       │       │   │   ├── 起始页码
│       │       │   │   ├── 结束页码
│       │       │   │   └── 电子书/音频资源（可选）
│       │       │   ├── 导读单任务
│       │       │   │   ├── 导读单资源
│       │       │   │   ├── 无答案版缩略图与文件名
│       │       │   │   ├── 有答案版缩略图与文件名
│       │       │   │   └── 分别解锁策略
│       │       │   ├── 视频任务
│       │       │   │   ├── 视频资源
│       │       │   │   └── 解锁策略
│       │       │   └── 测评任务
│       │       │       ├── 测评资源/题组
│       │       │       └── 解锁策略
│       │       └── 保存提示
│       └── 书籍配置状态
└── 统计数据
    ├── 报名班级数
    ├── 参与学校数
    ├── 参与学生数
    └── 整体完成率
```

**结构说明**

1. 一个阅读计划下可同时配置多个年级
2. 每个年级独立配置自己的书籍，不同年级之间书籍不共用
3. 同一本书在不同年级下视为独立配置对象，阅读开始时间、阅读次数、任务内容可不同
4. 阅读任务配置以“年级下的书籍”为单位进行，不以计划统一配置
5. 老师端和学生端实际看到的书目与任务，以所属班级年级对应的配置为准

#### 4.4.2 资源解锁机制

资源解锁策略支持以下三种类型：

1. **立即解锁**：学生报名后立即可以访问该资源
2. **定时解锁**：在指定的日期时间后解锁（如：2026-02-15 08:00）
3. **完成前置任务后解锁**：完成指定的前置任务后解锁（如：完成第1天打卡后解锁第2天视频）

## 5. 功能详情

### 5.1 登录与权限管理

#### 5.1.1 F001 账号密码登录

**功能描述**

支持平台管理人员使用工号和HR密码登录管理平台，外部人员使用手机号作为工号登录

**业务规则**

1. 账号规则：
   - 登录账号字段统一为工号
   - 内部人员输入本人工号
   - 外部人员没有独立工号时，使用本人手机号作为工号录入和登录
2. 密码规则：
   - 登录密码使用HR系统密码
   - 与公司其他平台账号密码体系通用
   - 管理平台不单独定义密码格式和修改规则
3. 登录成功后：
   - 生成登录token，有效期24小时
   - 记录登录日志（登录时间、IP地址、设备信息）
   - 跳转到首页

**页面元素**

| 元素名称           | 元素类型 | 必填 | 说明                                    |
| ------------------ | -------- | ---- | --------------------------------------- |
| 工号输入框         | 文本框   | 是   | 提示文字：请输入工号/手机号（外部人员） |
| 密码输入框         | 密码框   | 是   | 提示文字：请输入HR密码                  |
| 记住密码           | 复选框   | 否   | 勾选后7天内自动登录                     |
| 忘记密码链接       | 链接     | 否   | 点击提示"请联系管理员重置密码"          |
| 登录按钮           | 按钮     | -    | 点击提交登录请求                        |
| **交互说明** |          |      |                                         |

1. 输入工号和HR密码后，点击"登录"按钮
2. 系统验证工号和HR密码是否正确
3. 验证通过后，跳转到首页
4. 验证失败，显示错误提示：
   - 工号不存在：提示"工号不存在，请检查后重试"
   - 密码错误：提示"密码错误，请重新输入"
   - 账号被禁用：提示"账号已被禁用，请联系管理员"

**异常处理**

| 异常场景   | 处理方式                             |
| ---------- | ------------------------------------ |
| 网络异常   | 提示"网络连接失败，请检查网络后重试" |
| 服务器异常 | 提示"服务器异常，请稍后重试"         |
| 工号不存在 | 提示"工号不存在，请检查后重试"       |
| 密码错误   | 提示"密码错误，请重新输入"           |
| 账号被禁用 | 提示"账号已被禁用，请联系管理员"     |

**验收标准**

1. 输入正确的工号和HR密码，点击登录后成功跳转到首页
2. 输入错误密码，页面显示"密码错误，请重新输入"
3. 勾选"记住密码"后关闭浏览器，7天内再次访问可自动登录
4. 点击"忘记密码"显示"请联系管理员重置密码"提示
5. 网络断开时点击登录，显示"网络连接失败，请检查网络后重试"

#### 5.1.2 F002 企微单点登录

**功能描述**

支持平台管理人员从企业微信工作台点击管理平台入口，跳转至管理平台并自动完成登录，无需手动输入账号密码

**业务规则**

1. 企微工作台登录流程：
   - 管理平台作为企微工作台中的应用入口进行配置
   - 用户在企业微信工作台点击"管理平台"入口
   - 企业微信将身份凭证传递给管理平台
   - 管理平台基于身份凭证获取企微用户信息（姓名、工号、部门等）
   - 系统验证用户是否有管理平台权限
   - 验证通过后，自动登录并跳转到首页
2. 账号绑定规则：
   - 首次从企微工作台进入时，需要绑定已有的管理平台工号账号
   - 绑定弹窗中的账号字段统一显示为工号
   - 外部人员绑定时，输入手机号作为工号
   - 绑定后，后续再次从企微工作台进入时可直接自动登录
3. 登录成功后：
   - 生成登录token，有效期24小时
   - 记录登录日志（登录时间、IP地址、企微用户信息）
   - 跳转到首页

3. 系统获取企微用户信息并验证权限

   - 输入管理平台工号和HR密码

**异常处理**

**验收标准**

2. 首次从企微工作台进入时弹出账号绑定弹窗，输入正确工号和HR密码后绑定成功

##### 步骤2：配置阅读次数和任务

**业务规则**

1. 页面布局：
    - 左右两栏布局
    - 左侧显示步骤1已选的书籍列表，点击切换当前配置的书籍
    - 右侧显示当前书籍的阅读任务配置区域
2. 书籍列表（左栏）：
   - 显示书名、作者
   - 显示配置状态（"已配置 N 次阅读"或"未配置"）
   - 当前选中的书籍高亮显示
3. 阅读次数管理：
    - 点击"添加阅读次数"新增一次阅读
    - 每次阅读标题默认为"第N次阅读"，支持编辑
    - 每次阅读可单独删除
    - 页面顶部显示统计信息（共N次阅读，M个任务）
4. 每次阅读下通过"添加任务"维护任务列表，不预设固定任务组合
5. 可添加的任务类型包括：
   - 阅读任务：配置起始页码和结束页码，可关联书籍资源库中的电子书/音频资源
   - 导读单任务：选择一组导读单资源，资源内包含无答案版和有答案版文件
   - 观看视频：配置视频资源
   - 测评任务：配置测评资源/题组和解锁策略
   - 当前版本不限制同一阅读次数下同类型任务的添加次数，管理人员可按实际需要重复添加
6. 资源上传方式：
    - 阅读任务、导读单任务：从资源库选择，若资源不存在可前往资源库上传
    - 视频任务、测评任务：支持从资源库选择或直接本地上传新文件
    - 已选择的资源显示文件名，支持删除
7. 解锁策略：
   - 导读单任务需分别配置无答案版和有答案版的解锁策略，视频资源也需配置解锁策略
    - 任务当天解锁：学生到达该次阅读时解锁
    - 提前N天解锁：在任务日期前N天解锁（N为0-30的整数）
8. 测评任务配置：
   - 支持配置测评资源/题组
   - 解锁策略与其他任务一致，支持任务当天解锁或提前N天解锁（N为0-30的整数）
9. 任务日期兼容：
   - 当前版本管理人员不可手动配置开始日期和结束日期
   - 系统根据书籍开始时间和阅读顺序自动生成每次阅读的开始日期和结束日期
   - 当前版本默认开始日期与结束日期相同，即每次阅读默认对应1天
   - 后台需保存每次阅读的开始日期和结束日期，用于兼容后续区间周期版本
10. 测评任务提醒规则：
   - 若一本书未配置测评任务，在保存草稿、下一步、保存修改等操作时提示"当前未配置测评任务，请确认该书是否无需测评"
   - 若已配置测评任务但测评任务不在最后一次阅读中，在保存草稿、下一步、保存修改等操作时提示"当前测评任务未配置在最后一次阅读中，通常建议将测评放在最后一次阅读后，请确认是否符合本次计划安排"
   - 当前版本上述规则仅做提醒，不阻断操作

#### 5.4.5 F018 计划发布与撤回

**功能描述**

**业务规则**

1. 发布条件：
   - 计划状态为"未发布"
   - 至少配置了一个年级的书目
   - 每个已配置年级至少有一本书籍
   - 每本书至少配置了一次阅读任务
2. 发布流程：
   - 点击"发布"按钮，弹出确认弹窗
   - 确认弹窗提示"确认要发布此阅读计划吗？阅读计划发布后会显示在小程序界面上，老师能够看到并报名参与"

   - 成功提示提供"返回列表"和"查看详情"两个入口

   - 若当前时间早于计划开始时间，计划状态变为"待开始"
   - 到达计划开始时间后，计划状态自动变为"进行中"
   - 到达计划结束时间后，计划状态自动变为"已结束"
   - 老师端可见该计划并可报名
4. 撤回规则：
   - 待开始的计划可以撤回
   - 撤回后计划状态变为"未发布"
   - 已报名的班级将收到撤回通知
   - 进行中的计划不可撤回

| 撤回确认弹窗 | 弹窗     | 确认撤回操作                 |

**交互说明**

1. 在计划列表页，未发布的计划显示"发布"按钮
2. 点击"发布"弹出确认弹窗
3. 点击"确认发布"执行发布
4. 发布成功后弹出成功提示，可选择"返回列表"或"查看详情"
5. 待开始的计划显示"撤回"按钮
6. 点击"撤回"弹出确认弹窗，确认后执行撤回

**异常处理**

| 异常场景       | 处理方式                     |
| -------------- | ---------------------------- |
| 配置不完整     | 提示"请完善计划配置后再发布" |
| 发布失败       | 提示"发布失败，请重试"       |
| 撤回失败       | 提示"撤回失败，请重试"       |
| 计划进行中撤回 | 提示"进行中的计划不可撤回"   |

**验收标准**

1. 未发布且配置完整的计划，点击"发布"弹出确认弹窗，确认后发布成功
2. 发布成功后弹出成功提示，提供"返回列表"和"查看详情"两个入口
3. 配置不完整的计划点击"发布"，提示"请完善计划配置后再发布"
4. 待开始的计划，点击"撤回"弹出确认弹窗，确认后撤回成功，状态变为"未发布"
5. 进行中的计划，撤回按钮不可用或点击后提示"进行中的计划不可撤回"

3. 修改参数值后点击"保存"，提示保存成功，参数立即生效
4. 点击"取消"后本次修改不生效，页面恢复只读状态

#### 5.10.2 F043 数据字典

**列表字段**

| 字典分类 | 所属字典分类名称   |

| 字典值   | 字典项的显示值     |

| 状态     | 启用/禁用          |

**页面元素**

4. 编辑字典项后保存成功，数据正确更新
5. 禁用字典项后状态变为"禁用"，启用后恢复

#### 5.10.3 F044 AI导语配置

**功能描述**

配置学生端各场景下AI虚拟形象的引导语模板，支持参数标签动态替换

**业务规则**

1. 导语类型：
   - 当次阅读导语：学生进入当前阅读计划下当次阅读任务时的欢迎引导语
   - 观看视频：引导学生观看视频资源时的提示语
   - 阅读：引导学生进行阅读任务时的提示语
   - 导读单：引导学生完成导读单时的提示语
   - AI测评：引导学生进行测评时的提示语
2. 导语内容支持插入参数标签，运行时自动替换为实际值：
   - {书名}：当前阅读的书籍名称
   - {任务数量}：当前当次阅读待完成的任务数
   - {页码范围}：当前当次阅读的页码范围
   - {视频名称}：当前视频资源名称
   - {导读单名称}：当前导读单名称
3. 每条导语可设置启用/禁用状态，禁用后不再展示给学生
4. 同一类型可配置多条导语，系统随机展示启用状态的导语

**列表字段**

| 字段名称 | 字段说明                             |
| -------- | ------------------------------------ |
| 序号     | 自动编号                             |
| 导语类型 | 当次阅读导语/观看视频/阅读/导读单/AI测评 |
| 导语内容 | 导语文本内容，含参数标签             |
| 状态     | 启用/禁用                            |
| 更新时间 | 最近修改时间                         |
| 更新人   | 最近修改的操作人                     |

**页面元素**

| 元素名称     | 元素类型 | 说明                         |
| ------------ | -------- | ---------------------------- |
| 筛选区域     | 筛选栏   | 按导语类型、状态、关键词筛选 |
| 新增导语按钮 | 按钮     | 打开新增导语弹窗             |
| 编辑按钮     | 按钮     | 编辑选中的导语               |
| 删除按钮     | 按钮     | 删除选中的导语               |
| 禁用按钮     | 按钮     | 禁用选中的导语               |
| 启用按钮     | 按钮     | 启用选中的导语               |
| 导语列表     | 表格     | 展示所有导语记录             |

**交互说明**

1. 进入AI导语配置页面，显示导语列表
2. 支持按导语类型、状态筛选，支持关键词搜索
3. 点击"新增导语"弹出弹窗，选择导语类型、输入导语内容、设置状态后保存
4. 导语内容编辑时，点击下方参数标签可插入到内容中
5. 点击"编辑"可修改已有导语
6. 点击"删除"弹出确认弹窗，确认后删除
7. 点击"禁用"弹出确认弹窗，确认后切换为禁用状态
8. 点击"启用"弹出确认弹窗，确认后切换为启用状态

**异常处理**

| 异常场景       | 处理方式               |
| -------------- | ---------------------- |
| 导语内容为空   | 提示"请输入导语内容"   |
| 未选择导语类型 | 提示"请选择导语类型"   |
| 保存失败       | 提示"保存失败，请重试" |

**验收标准**

1. 进入AI导语配置页面，列表展示导语类型、导语内容、状态、更新时间等字段
2. 按导语类型、状态筛选，列表正确过滤
3. 新增导语时选择类型、输入内容、点击参数标签可插入到内容中，保存成功
4. 编辑已有导语后保存成功，数据正确更新
5. 禁用/启用导语后状态正确切换
6. 删除导语弹出确认弹窗，确认后删除成功
