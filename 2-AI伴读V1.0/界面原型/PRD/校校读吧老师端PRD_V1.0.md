## 2. 背景与目标

### 2.1 项目背景

随着教育信息化的深入推进，学校对学生阅读能力培养的重视程度不断提升。传统的阅读教学存在以下问题：

- 教师难以追踪每位学生的阅读进度和效果
- 缺乏有效的阅读激励机制
- 阅读数据分散，无法形成系统性分析
- 家校沟通不畅，家长难以参与学生阅读管理

**现状痛点：**

- 教师无法实时了解学生阅读进度，只能依赖定期检查
- 阅读任务布置和跟踪依赖纸质材料，效率低下
- 缺乏科学的阅读测评体系，难以量化阅读效果
- 班级阅读数据分散，无法进行横向对比分析

**影响范围：**

- 影响全校语文教师的阅读教学工作
- 影响学生阅读习惯的养成和阅读能力的提升

"校校读吧"平台应运而生，旨在通过数字化手段提升学校阅读教学效率，本文档聚焦于老师端产品的功能需求

### 2.2 产品目标

| 目标类型 | 具体目标                     | 衡量指标            | 基准值       | 统计周期    |
| -------- | ---------------------------- | ------------------- | ------------ | ----------- |
| 业务目标 | 提升学校阅读教学数字化水平   | 教师使用率≥80%     | 0%           | 上线后3个月 |
| 用户目标 | 帮助教师高效管理班级阅读活动 | 任务处理效率提升50% | 传统纸质方式 | 上线后1个月 |
| 技术目标 | 提供稳定流畅的移动端体验     | 页面加载≤2秒       | -            | 持续监控    |

**指标说明：**

| 指标         | 计算方式                            | 数据来源     |
| ------------ | ----------------------------------- | ------------ |
| 教师使用率   | 月活跃教师数 / 已注册教师数 × 100% | 后台数据统计 |
| 任务处理效率 | 对比使用系统前后完成相同任务的时间  | 用户调研问卷 |
| 页面加载时间 | 首屏内容完全渲染的时间（P95）       | 性能监控系统 |

### 2.3 项目范围

**本期包含：**

- 教师登录与身份认证
- 班级创建与学生管理
- 阅读计划报名与发布
- 学生阅读进度监控
- 测评管理与成绩查看
- 数据报告与导出
- 消息推送功能
- 个人中心设置

**本期不包含：**

- 自定义阅读计划创建 - 原因：后续版本迭代
- 家长端功能 - 原因：独立产品线
- 积分商城兑换 - 原因：运营功能后续开发

> 注：积分功能本期仅包含积分查看（F015积分明细），不包含积分兑换功能。教师可查看积分获取记录，但暂不支持积分消费

## 3. 用户分析

### 3.1 目标用户

| 用户角色 | 特征描述                   | 核心诉求             | 使用场景           |
| -------- | -------------------------- | -------------------- | ------------------ |
| 语文教师 | 负责班级阅读教学的一线教师 | 高效管理学生阅读任务 | 课间、办公室、家中 |
| 班主任   | 负责班级整体管理的教师     | 了解班级阅读整体情况 | 班会、家长会准备   |
| 教研组长 | 负责学科教研的骨干教师     | 查看年级阅读数据对比 | 教研会议、数据分析 |

> 注：教研组长为特殊权限角色，需学校管理员在后台系统中配置后方可查看年级维度数据，详见5.2安全需求中的数据权限说明

### 3.2 用户画像

**典型用户1：王老师（语文教师）**

- 基本信息：35岁，初中语文教师，教龄10年
- 行为特征：习惯使用手机处理工作事务，每天查看学生作业情况
- 痛点：管理3个班级150多名学生的阅读任务，工作量大
- 期望：能够快速了解学生阅读进度，及时发现落后学生并提醒

**典型用户2：李老师（班主任）**

- 基本信息：40岁，小学班主任兼语文教师，教龄15年
- 行为特征：注重班级整体表现，定期与家长沟通学生情况
- 痛点：家长会前需要手动整理学生阅读数据，耗时费力
- 期望：一键导出班级阅读报告，直观展示班级阅读成果

**典型用户3：张老师（教研组长）**

- 基本信息：45岁，初中语文教研组长，教龄20年
- 行为特征：关注年级整体教学质量，定期组织教研活动
- 痛点：缺乏跨班级的阅读数据对比，难以发现教学问题
- 期望：查看年级各班阅读数据对比，为教研决策提供依据

### 3.3 用户旅程

**场景：教师为班级报名阅读计划并监控学生进度**

| 阶段               | 登录系统   | 查看阅读计划 | 为班级报名     | 监控学生进度   | 查看测评成绩 | 导出数据报告 |
| ------------------ | ---------- | ------------ | -------------- | -------------- | ------------ | ------------ |
| **用户情绪** | 期待       | 好奇         | 满意           | 关注           | 欣慰/担忧    | 成就感       |
| **痛点1**    | 登录繁琐   | 计划太多     | 班级多选麻烦   | 数据更新不及时 | 成绩解读困难 | 导出格式单一 |
| **痛点2**    | 记不住密码 | 不知如何选择 | 不确定是否成功 | 找不到落后学生 | 缺乏改进建议 | 数据不够详细 |

**用户旅程流程图：**

```
[登录系统] ──▶ [查看阅读计划] ──▶ [为班级报名] ──▶ [监控学生进度] ──▶ [查看测评成绩] ──▶ [导出数据报告]
```

**关键触点优化：**

| 阶段         | 用户期望         | 设计要点                       |
| ------------ | ---------------- | ------------------------------ |
| 登录系统     | 快速进入         | 支持微信一键登录，记住登录状态 |
| 查看阅读计划 | 快速找到合适计划 | 清晰的状态标签，支持搜索筛选   |
| 为班级报名   | 操作简单明确     | 支持批量选择，实时反馈报名结果 |
| 监控学生进度 | 一目了然         | 可视化进度条，支持排序筛选     |
| 查看测评成绩 | 了解学生表现     | AI智能评价，多维度能力分析     |
| 导出数据报告 | 方便汇报使用     | 多格式支持，数据完整准确       |

### 4.5 功能详情

#### F001 微信登录

**线框图**

```
┌─────────────────────────────────┐
│         01-登录页面              │
├─────────────────────────────────┤
│                                 │
│      ┌─────────────────┐        │
│      │                 │        │
│      │   校校读吧LOGO   │        │
│      │                 │        │
│      └─────────────────┘        │
│                                 │
│         校校读吧老师端           │
│                                 │
│    ┌───────────────────────┐    │
│    │ [微信登录] [手机号登录] │    │
│    └───────────────────────┘    │
│                                 │
│      ┌─────────────────┐        │
│      │  🟢 微信一键登录  │        │
│      └─────────────────┘        │
│                                 │
│    首次使用？联系管理员          │
│                                 │
│  登录即表示同意《用户协议》      │
│  和《隐私政策》                  │
│                                 │
└─────────────────────────────────┘
```

**功能说明**

1. 教师通过微信授权快速登录系统，无需输入账号密码
2. 首次登录需要微信授权获取用户信息（头像、昵称）
