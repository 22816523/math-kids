# 20以内进位加法 - 竞品教学设计分析

**分析日期**: 2026-03-12
**分析目标**: 优化我们的Level 2（20以内进位加法）教学设计
**参考竞品**: Teachley Addimals EDU, MathEdge Addition, KidloLand Math Academy

---

## 1. 核心教学方法：凑十法（Making Ten Strategy）

### 1.1 什么是凑十法

20以内进位加法的本质是"凑十法"：
- **例子**: 9 + 5 = ?
- **思路**: 9 + 1 = 10，5 - 1 = 4，所以 10 + 4 = 14
- **口诀**: "拆小数，凑大数，先凑十，再相加"

### 1.2 为什么要用凑十法

- 符合儿童认知发展规律（3-8岁）
- 10是进位制的基础，建立"满十进一"概念
- 可视化程度高，容易理解

## 2. 可视化工具：十框（Ten Frame）

### 2.1 十框的标准形式

```
┌─┬─┬─┬─┬─┐
│●│●│●│●│●│  ← 第一行5个
├─┼─┼─┼─┼─┤
│●│●│●│●│○│  ← 第二行5个（共10个格子）
└─┴─┴─┴─┴─┘
```

### 2.2 竞品使用方式

**Teachley Addimals EDU** 的做法：
- 使用2个十框：一个放第一个数，一个放第二个数
- 通过动画展示"移动"过程，把第二个框的部分数字移到第一个框凑成10
- 最后显示"1个满框（10）+ 剩余数字"

**MathEdge Addition** 的做法：
- 使用1个十框 + 额外区域
- 先填充第一个数（如9个）
- 再逐个添加第二个数，当填满10个时有特殊动画提示
- 剩余的数字放在框外

## 3. 分步骤展示进位过程

### 3.1 标准教学流程（以 9+5 为例）

**步骤1：展示问题**
- 显示算式：9 + 5 = ?
- 在十框中显示9个圆点（填满第一行5个 + 第二行4个）

**步骤2：引入第二个数**
- 在十框旁边显示5个圆点（待添加）
- 提示："我们要加5个"

**步骤3：凑十（关键步骤）**
- 高亮显示：十框还差1个就满了
- 从5个圆点中，1个移动到十框中
- 动画效果：移动过程 + 满框提示音/动画
- 提示："9加1等于10，满了"

**步骤4：显示剩余**
- 十框已满（10个）
- 外面还剩4个圆点
- 提示："还剩4个"

**步骤5：得出答案**
- 显示：10 + 4 = 14
- 完整过程：9 + 5 = 9 + 1 + 4 = 10 + 4 = 14

### 3.2 竞品的分步实现

**Teachley Addimals EDU**：
- 每一步都需要儿童点击"下一步"按钮
- 有语音讲解每个步骤
- 关键步骤有动画强化（如满10时的庆祝动画）

**MathEdge Addition**：
- 自动播放动画，但可以暂停/重播
- 有"慢动作"模式，让儿童看清每个数字的移动
- 配合语音："让我们把1个移过去，凑成10"

## 4. 交互设计：演示 vs 操作

### 4.1 演示模式（Demo）

**目的**：让儿童理解"凑十"的概念

**交互方式**：
- 儿童主动触发每一步（点击"凑十"按钮、点击要移动的圆点等）
- 不是被动观看，而是参与操作
- 操作错误时有提示，但不会失败

**示例**：
- 显示9+5，十框中有9个圆点，旁边有5个
- 提示："点击1个圆点，把它移到框里凑成10"
- 儿童点击后，圆点移动，框满了
- 提示："太好了！现在是10加4"

### 4.2 练习模式（Practice）

**目的**：检验儿童是否掌握

**交互方式**：
- 直接给出算式和选项
- 不展示凑十过程（除非点击"看演示"）
- 答对/答错有即时反馈

**关键**：
- 演示和练习必须分开
- 演示是教学，练习是检验
- 演示可以反复看，练习有次数限制

## 5. 当前实现的问题

### 5.1 概念错误

```javascript
// 当前代码（错误）
for (let i = 0; i < a; i++) {
  tensCells[i].classList.add('filled');  // 把9放在"十位"框
}
for (let i = 0; i < b; i++) {
  onesCells[i].classList.add('gray');    // 把5放在"个位"框
}
```

**问题**：
- 9应该在"个位"框，不是"十位"框
- 这不是位值概念（place value），而是凑十概念（making ten）
- 用两个十框容易混淆"十位/个位"和"凑十"

### 5.2 缺少关键步骤

- 没有展示"从5中拿1个凑成10"的过程
- 直接跳到结果，儿童不理解为什么是14
- "满十进一"按钮的含义不清楚

## 6. 设计建议

### 6.1 方案A：单十框 + 外部区域（推荐）

**布局**：
```
        9 + 5 = ?

   ┌─────────┐
   │ 十框区域 │  ← 最多10个
   │ ●●●●●   │
   │ ●●●●    │
   └─────────┘

   [ ○○○○○ ]  ← 待添加区域
```

**演示流程**：
1. 十框中显示9个圆点
2. 下方显示5个灰色圆点（待添加）
3. 提示："点击1个圆点移到框里"
4. 儿童点击后，1个移入框中，框满了（特效）
5. 显示："10 + 4 = 14"

**优点**：
- 概念清晰，只有一个框
- 直观展示"凑十"过程
- 符合竞品最佳实践

### 6.2 方案B：双十框（不推荐）

保留当前的双十框，但修正逻辑：
- 左框：放第一个数（9个）
- 右框：放第二个数（5个）
- 动画：从右框移1个到左框

**缺点**：
- 容易和"十位/个位"概念混淆
- 需要更多空间
- 不如单框直观

### 6.3 具体实现建议

**演示模式**：
```javascript
function showLevel2Demo() {
  // 步骤1：显示9个在十框中
  // 步骤2：显示5个在待添加区
  // 步骤3：提示点击1个移入框
  // 步骤4：框满了，特效
  // 步骤5：显示10+4=14
}
```

**练习模式**：
- 保持当前的4选项答题方式
- "看演示"按钮触发上述演示

## 7. 实施优先级

### 高优先级（必须改）
1. 修正概念错误：不要用"十位/个位"标签
2. 改用单十框方案
3. 添加分步骤演示

### 中优先级（建议改）
1. 添加动画效果（移动、满框特效）
2. 添加语音提示（可选）
3. 优化视觉反馈

### 低优先级（可以后续优化）
1. 多种可视化元素（不只是圆点，可以是水果等）
2. 慢动作模式
3. 成就系统
