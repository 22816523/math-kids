# 校校读吧-老师端 产品需求文档

## 文档信息

| 项目       | 内容                                  |
| ---------- | ------------------------------------- |
| 产品名称   | 校校读吧-老师端                       |
| 文档版本   | V1.0                                  |
| 产品定位   | 面向中小学教师的阅读管理工具          |

## 一、术语定义

### 1.1 业务术语

| 术语     | 定义                                                                         |
| -------- | ---------------------------------------------------------------------------- |
| 阅读计划 | 平台发布的季度性阅读活动，包含多本书籍和对应的阅读任务，教师可为班级报名参与 |
| 书籍关卡 | 阅读计划中每本书籍的阅读任务单元，学生需按关卡顺序完成阅读和测评             |
| 打卡     | 学生完成每日阅读任务后的签到记录，用于追踪阅读进度                           |
| 测评     | 针对书籍内容设置的理解测试题目，用于检验学生阅读效果                         |
| 报名     | 教师为班级加入某个阅读计划的操作，报名后班级学生可参与该计划                 |
| 进度监控 | 教师查看班级学生阅读完成情况的功能模块                                       |
| 班级     | 教师管理的学生群组单位，以年级+班号命名                                      |
| 学段     | 教育阶段划分，包括小学、初中、高中                                           |
| 积分     | 用户在平台活动中获得的奖励点数，可用于兑换或排名                             |

### 1.2 技术术语

| 术语     | 定义                                               |
| -------- | -------------------------------------------------- |
| 本地缓存 | 指移动端沙盒存储，用于离线数据暂存，非浏览器Cookie |
| 同步     | 指本地数据与服务端数据的双向一致性操作             |
| 幂等性   | 同一请求多次执行与单次执行效果相同，避免重复数据   |
| DOM节点  | HTML文档中的元素对象，通过ID或类名定位             |
| 事件绑定 | 将用户操作（如点击、输入）与JavaScript函数关联     |

### 1.3 状态定义

| 状态名称 | 适用对象      | 含义                               |
| -------- | ------------- | ---------------------------------- |
| 未开始   | 阅读计划      | 计划开始日期未到，不可报名         |
| 进行中   | 阅读计划      | 计划在有效期内，可报名和参与       |
| 已结束   | 阅读计划      | 计划结束日期已过，仅可查看历史数据 |
| 未报名   | 班级-计划关系 | 班级尚未加入该阅读计划             |
| 已报名   | 班级-计划关系 | 班级已加入该阅读计划，学生可参与   |
| 已完成   | 打卡/测评     | 学生已完成该任务项                 |
| 未完成   | 打卡/测评     | 学生尚未完成该任务项               |

### 3.1 用户认证模块

#### 3.1.1 登录页（/老师端/01-登录.html）

**页面说明**：教师登录入口页面，支持微信授权登录和手机验证码登录两种方式，用于身份验证和获取系统访问权限

**功能清单**

| 功能点         | DOM节点      | 绑定事件                     | 功能作用                                                 |
| -------------- | ------------ | ---------------------------- | -------------------------------------------------------- |
| 登录方式切换   | .tab-item    | click→switchTab()           | 切换微信登录和手机号登录两种方式，满足不同用户的登录习惯 |
| 微信一键登录   | .wechat-btn  | click→wechatLogin()         | 调用微信授权接口完成快捷登录，减少用户输入成本           |
| 手机号输入     | #phoneInput  | input                        | 接收用户输入的11位手机号，用于验证码登录身份识别         |
| 验证码输入     | #codeInput   | input                        | 接收用户输入的6位验证码，用于验证手机号归属              |
| 获取验证码     | #getCodeBtn  | click→getVerificationCode() | 向用户手机发送验证码，启动60秒倒计时防止频繁请求         |
| 验证码登录提交 | .login-btn   | click→handleCodeLogin()     | 校验手机号和验证码，完成登录并跳转至首页                 |
| 联系管理员     | .footer-link | click→alert()               | 引导首次使用的教师联系管理员开通账号                     |
| 用户协议链接   | .agreement a | click                        | 跳转至用户协议页面，满足合规要求                         |

**交互逻辑**

| 功能点       | 触发条件                                           | 操作流程                                                                          | 状态反馈                                                     | 异常场景                                           | 异常处理                                                                       |
| ------------ | -------------------------------------------------- | --------------------------------------------------------------------------------- | ------------------------------------------------------------ | -------------------------------------------------- | ------------------------------------------------------------------------------ |
| 登录方式切换 | 点击.tab-item元素                                  | 1.更新Tab样式active状态 2.切换对应.tab-content显示 3.清空表单内容                 | 当前Tab高亮，对应内容区显示                                  | 无                                                 | 无                                                                             |
| 微信一键登录 | 点击.wechat-btn按钮                                | 1.调用微信授权接口 2.获取用户信息 3.写入localStorage 4.跳转至06-阅读计划列表.html | 成功：弹出"登录成功"提示后跳转；加载中：按钮显示加载状态     | 微信授权失败、网络异常                             | 弹出错误提示，保持当前页面                                                     |
| 获取验证码   | 点击#getCodeBtn且手机号格式正确（/^1[3-9]\d{9}$/） | 1.校验手机号格式 2.调用发送验证码接口 3.启动60秒倒计时 4.更新按钮文案             | 成功：弹出"验证码已发送"，按钮显示倒计时；失败：显示错误提示 | 手机号格式错误、发送频率限制、网络异常             | 格式错误：#phoneInput添加.error类，显示#phoneError；频率限制：提示"请稍后再试" |
| 验证码登录   | 点击.login-btn且表单校验通过                       | 1.校验手机号格式 2.校验验证码长度=6 3.调用登录接口 4.写入localStorage 5.跳转首页  | 成功：弹出"登录成功"后跳转；失败：显示具体错误               | 手机号格式错误、验证码错误、验证码过期、账号不存在 | 对应输入框添加.error类，显示错误提示文案                                       |

**接口说明**

| 接口名称   | 请求路径                     | 请求方式 | 入参                                      | 出参                                           | 错误码                                                |
| ---------- | ---------------------------- | -------- | ----------------------------------------- | ---------------------------------------------- | ----------------------------------------------------- |
| 发送验证码 | POST /api/auth/sendCode      | POST     | phone:string(必填,11位手机号)             | success:boolean, message:string                | 2001:手机号格式错误 2002:发送频率限制 1001:服务器错误 |
| 验证码登录 | POST /api/auth/loginByCode   | POST     | phone:string(必填), code:string(必填,6位) | success:boolean, token:string, userInfo:object | 2003:验证码错误 2004:验证码过期 2005:账号不存在       |
| 微信登录   | POST /api/auth/loginByWechat | POST     | code:string(微信授权码)                   | success:boolean, token:string, userInfo:object | 2006:微信授权失败 2007:账号未绑定                     |

**数据存储**

- 登录成功后将isLoggedIn='true'写入localStorage
- 用户信息（name、account、title）以JSON格式存入localStorage的userInfo字段
- Token存入localStorage用于后续接口鉴权

### 3.2 积分与活动模块

| 底部导航-进度 | .tab-item:nth(2) | click→Router.toTeacherPage('monitor')  | 跳转至进度监控页                                   |
| 底部导航-我的 | .tab-item:last   | click                                   | 跳转至教师个人中心页（当前高亮）                   |

| 功能点     | 触发条件                         | 操作流程                                                                          | 状态反馈                                                     | 异常场景            | 异常处理                                      |
| ---------- | -------------------------------- | --------------------------------------------------------------------------------- | ------------------------------------------------------------ | ------------------- | --------------------------------------------- |

**接口说明**

| 接口名称     | 请求路径                | 请求方式 | 入参                                        | 出参                                                        | 错误码                         |
| ------------ | ----------------------- | -------- | ------------------------------------------- | ----------------------------------------------------------- | ------------------------------ |
| 获取积分明细 | GET /api/teacher/points | GET      | page:number(默认1), pageSize:number(默认20) | totalPoints:number, records:array[{title,desc,amount,time}] | 2000:Token无效 1001:服务器错误 |

**数据存储**

- 积分数据从服务端实时获取，不做本地缓存
- 页面生命周期：进入页面时调用接口加载数据

## 附录

### A. 错误码定义规范

| 错误码范围 | 类型     | 说明                            |
| ---------- | -------- | ------------------------------- |
| 1000-1999  | 系统错误 | 服务器内部错误、网络超时等      |
| 2000-2999  | 认证错误 | 登录失败、Token过期、权限不足等 |
| 3000-3999  | 业务错误 | 数据校验失败、业务规则冲突等    |
| 4000-4999  | 数据错误 | 数据不存在、数据格式错误等      |

### B. 通用交互规范

| 场景         | 处理方式                       |
| ------------ | ------------------------------ |
| 网络请求中   | 显示加载状态，禁用操作按钮     |
| 网络请求成功 | 移除加载状态，更新页面数据     |
| 网络请求失败 | 显示错误提示，提供重试入口     |
| 断网状态     | 显示断网提示，读取本地缓存数据 |
| 弱网状态     | 延长超时时间，显示加载进度     |
| 空数据       | 显示空状态占位图和引导文案     |

### C. 数据存储规范

| 数据类型 | 存储位置        | 有效期  | 同步策略               |
| -------- | --------------- | ------- | ---------------------- |
| 登录状态 | localStorage    | 长期    | 登录时写入，退出时清除 |
| 用户信息 | localStorage    | 长期    | 登录时写入，编辑时更新 |
| 班级数据 | 服务端+本地缓存 | 本地7天 | 页面加载时同步         |
| 阅读计划 | 服务端+本地缓存 | 本地7天 | 页面加载时同步         |
| 学生进度 | 服务端          | 永久    | 实时同步               |
