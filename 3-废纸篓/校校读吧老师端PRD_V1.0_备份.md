## 2. 背景与目标

### 2.1 项目背景

随着教育信息化的深入推进，学校对学生阅读能力培养的重视程度不断提升。传统的阅读教学存在以下问题：

- 教师难以追踪每位学生的阅读进度和效果
- 缺乏有效的阅读激励机制
- 阅读数据分散，无法形成系统性分析
- 家校沟通不畅，家长难以参与学生阅读管理

"校校读吧"平台应运而生，旨在通过数字化手段提升学校阅读教学效率，本文档聚焦于教师端产品的功能需求

### 2.2 问题陈述

**现状问题：**

- 问题1：教师无法实时了解学生阅读进度，只能依赖定期检查
- 问题2：阅读任务布置和跟踪依赖纸质材料，效率低下
- 问题3：缺乏科学的阅读测评体系，难以量化阅读效果
- 问题4：班级阅读数据分散，无法进行横向对比分析

**影响范围：**

- 影响全校语文教师的阅读教学工作
- 影响学生阅读习惯的养成和阅读能力的提升

### 2.3 产品目标

| 目标类型 | 具体目标                     | 衡量指标            | 基准值       | 统计周期    |
| -------- | ---------------------------- | ------------------- | ------------ | ----------- |
| 业务目标 | 提升学校阅读教学数字化水平   | 教师使用率≥80%     | 0%           | 上线后3个月 |
| 用户目标 | 帮助教师高效管理班级阅读活动 | 任务处理效率提升50% | 传统纸质方式 | 上线后1个月 |
| 技术目标 | 提供稳定流畅的移动端体验     | 页面加载≤2秒       | -            | 持续监控    |

**指标说明：**

| 指标         | 计算方式                            | 数据来源     |
| ------------ | ----------------------------------- | ------------ |
| 教师使用率   | 月活跃教师数 / 已注册教师数 × 100% | 后台数据统计 |
| 任务处理效率 | 对比使用系统前后完成相同任务的时间  | 用户调研问卷 |
| 页面加载时间 | 首屏内容完全渲染的时间（P95）       | 性能监控系统 |

### 2.4 项目范围

**本期包含：**

- 教师登录与身份认证
- 班级创建与学生管理
- 阅读计划报名与发布
- 学生阅读进度监控
- 测评管理与成绩查看
- 数据报告与导出
- 消息推送功能
- 个人中心设置

**本期不包含：**

- 自定义阅读计划创建 - 原因：后续版本迭代
- 家长端功能 - 原因：独立产品线
- 积分商城兑换 - 原因：运营功能后续开发

## 3. 用户分析

### 3.1 目标用户

| 用户角色 | 特征描述                   | 核心诉求             | 使用场景           |
| -------- | -------------------------- | -------------------- | ------------------ |
| 语文教师 | 负责班级阅读教学的一线教师 | 高效管理学生阅读任务 | 课间、办公室、家中 |
| 班主任   | 负责班级整体管理的教师     | 了解班级阅读整体情况 | 班会、家长会准备   |
| 教研组长 | 负责学科教研的骨干教师     | 查看年级阅读数据对比 | 教研会议、数据分析 |

### 3.2 用户画像

**典型用户1：王老师（语文教师）**

- 基本信息：35岁，初中语文教师，教龄10年
- 行为特征：习惯使用手机处理工作事务，每天查看学生作业情况
- 痛点：管理3个班级150多名学生的阅读任务，工作量大
- 期望：能够快速了解学生阅读进度，及时发现落后学生并提醒

**典型用户2：李老师（班主任）**

- 基本信息：40岁，小学班主任兼语文教师，教龄15年
- 行为特征：注重班级整体表现，定期与家长沟通学生情况
- 痛点：家长会前需要手动整理学生阅读数据，耗时费力
- 期望：一键导出班级阅读报告，直观展示班级阅读成果

**典型用户3：张老师（教研组长）**

- 基本信息：45岁，初中语文教研组长，教龄20年
- 行为特征：关注年级整体教学质量，定期组织教研活动
- 痛点：缺乏跨班级的阅读数据对比，难以发现教学问题
- 期望：查看年级各班阅读数据对比，为教研决策提供依据

### 3.3 用户旅程

**场景：教师为班级报名阅读计划并监控学生进度**

```
[登录系统] → [查看阅读计划] → [为班级报名] → [监控学生进度] → [查看测评成绩] → [导出数据报告]
     ↓              ↓               ↓               ↓                ↓               ↓
   情绪：          情绪：           情绪：          情绪：            情绪：          情绪：
   期待           好奇             满意            关注              欣慰/担忧        成就感
     ↓              ↓               ↓               ↓                ↓               ↓
   痛点：          痛点：           痛点：          痛点：            痛点：          痛点：
   登录繁琐       计划太多         班级多选麻烦    数据更新不及时    成绩解读困难    导出格式单一
   记不住密码     不知如何选择     不确定是否成功  找不到落后学生    缺乏改进建议    数据不够详细
```

**关键触点优化：**

| 阶段         | 用户期望         | 设计要点                       |
| ------------ | ---------------- | ------------------------------ |
| 登录系统     | 快速进入         | 支持微信一键登录，记住登录状态 |
| 查看阅读计划 | 快速找到合适计划 | 清晰的状态标签，支持搜索筛选   |
| 为班级报名   | 操作简单明确     | 支持批量选择，实时反馈报名结果 |
| 监控学生进度 | 一目了然         | 可视化进度条，支持排序筛选     |
| 查看测评成绩 | 了解学生表现     | AI智能评价，多维度能力分析     |
| 导出数据报告 | 方便汇报使用     | 多格式支持，数据完整准确       |

### 4.3 功能详情

#### F001 微信登录

**功能描述：**
教师通过微信授权快速登录系统，无需输入账号密码

**用户故事：**
作为教师，我希望通过微信一键登录，以便快速进入系统开始工作

**业务规则：**

1. 首次登录需要微信授权获取用户信息
2. 授权成功后自动创建账号并登录
3. 登录状态保存在本地存储中

**界面要素：**

| 元素         | 类型 | 说明                       |
| ------------ | ---- | -------------------------- |
| 微信图标     | 图标 | 绿色微信品牌图标           |
| 微信一键登录 | 按钮 | 绿色背景，点击触发微信授权 |
| 用户协议     | 链接 | 底部协议文案               |

**验收标准：**

- [ ] 点击按钮后调起微信授权
- [ ] 授权成功后跳转到阅读计划列表页
- [ ] 登录状态持久化保存

**异常处理：**

| 异常场景     | 处理方式  | 提示文案                         |
| ------------ | --------- | -------------------------------- |
| 用户拒绝授权 | 弹窗提示  | 需要微信授权才能登录，请重新授权 |
| 网络异常     | Toast提示 | 网络连接失败，请检查网络后重试   |
| 微信版本过低 | 弹窗提示  | 请升级微信版本后重试             |
| 服务器异常   | Toast提示 | 服务繁忙，请稍后重试             |
| 账号被禁用   | 弹窗提示  | 账号已被禁用，请联系管理员       |

**首次登录引导：**

新用户首次登录成功后，系统自动进入引导流程：

| 步骤 | 引导内容     | 是否必填 | 说明                     |
| ---- | ------------ | -------- | ------------------------ |
| 1    | 完善基本信息 | 是       | 填写姓名、选择学段、学科 |
| 2    | 创建首个班级 | 否       | 引导创建班级，可跳过     |
| 3    | 功能介绍     | 否       | 简要介绍核心功能，可跳过 |

引导完成后进入阅读计划列表页

#### F002 手机验证码登录

**功能描述：**
教师通过手机号和短信验证码登录系统

**业务规则：**

1. 手机号格式校验：11位数字，以1开头
2. 验证码为6位数字
3. 验证码有效期5分钟
4. 验证码发送后60秒内不可重复获取（重发冷却）
5. 同一手机号每日最多发送10条验证码
6. 验证码连续输入错误5次后，锁定15分钟
7. 验证码输入错误提示"验证码错误，请重新输入"

**界面要素：**

| 元素         | 类型   | 说明         | 校验规则     |
| ------------ | ------ | ------------ | ------------ |
| 手机号输入框 | 输入框 | 带手机图标   | 11位数字     |
| 验证码输入框 | 输入框 | 带盾牌图标   | 6位数字      |
| 获取验证码   | 按钮   | 倒计时显示   | 60秒冷却     |
| 登录         | 按钮   | 蓝色渐变背景 | 表单校验通过 |

**异常处理：**

| 异常场景         | 处理方式   | 提示文案                           |
| ---------------- | ---------- | ---------------------------------- |
| 手机号格式错误   | 输入框标红 | 请输入正确的11位手机号             |
| 验证码格式错误   | 输入框标红 | 请输入6位验证码                    |
| 验证码错误       | 输入框标红 | 验证码错误，请重新输入             |
| 验证码已过期     | 弹窗提示   | 验证码已过期，请重新获取           |
| 账号不存在       | 弹窗提示   | 请联系管理员开通账号               |
| 发送频率超限     | Toast提示  | 发送过于频繁，请稍后再试           |
| 每日发送次数超限 | Toast提示  | 今日验证码发送次数已达上限         |
| 错误次数超限     | 弹窗提示   | 验证码错误次数过多，请15分钟后重试 |

#### F003 阅读计划列表

**功能描述：**
展示平台发布的阅读计划，支持搜索和筛选，教师可查看计划详情或报名

**业务规则：**

1. 默认显示进行中和未开始的计划，已结束计划在"历史活动"入口查看
2. 计划状态分为：未开始、进行中、已结束
3. 已报名的计划显示"已报名X个班级"标签
4. 未开始的计划显示开始日期提示，不可报名

**界面要素：**

| 元素         | 类型   | 说明                         |
| ------------ | ------ | ---------------------------- |
| 搜索框       | 输入框 | 搜索活动名称                 |
| 计划卡片     | 卡片   | 显示计划名称、时间、书籍数量 |
| 状态标签     | 标签   | 进行中/未开始                |
| 立即报名     | 按钮   | 进行中且未报名时显示         |
| 历史活动入口 | 链接   | 底部显示已结束计划数量       |

**操作流程：**

1. 用户进入页面，系统加载计划列表
2. 用户可通过搜索框筛选计划
3. 点击计划卡片进入详情预览
4. 点击"立即报名"进入报名页面

**验收标准：**

- [ ] 页面加载时显示进行中和未开始的计划
- [ ] 搜索框输入关键词后实时筛选计划列表
- [ ] 已报名计划显示"已报名X个班级"标签
- [ ] 未开始计划的报名按钮显示为禁用状态
- [ ] 点击历史活动入口可查看已结束计划
- [ ] 下拉刷新可重新加载计划列表

**异常处理：**

| 异常场景     | 处理方式   | 提示文案                 |
| ------------ | ---------- | ------------------------ |
| 计划列表为空 | 显示空状态 | 暂无阅读计划，请稍后查看 |
| 搜索无结果   | 显示空状态 | 未找到相关计划           |
| 加载失败     | Toast提示  | 加载失败，请下拉刷新重试 |

#### F004 计划详情预览

**功能描述：**
展示阅读计划的详细信息，包括计划简介、书籍列表、关卡设置等

**业务规则：**

1. 显示计划名称、时间范围、书籍数量
2. 书籍按阅读顺序排列，显示序号徽章
3. 点击书籍卡片可查看关卡详情
4. 底部操作栏根据计划状态显示不同按钮

**界面要素：**

### 9.3 通用交互规范

**导航栏：**

**弹窗类型：**

- Modal弹窗：居中显示，半透明遮罩
- 底部弹窗：从底部滑出，圆角顶部
- Toast提示：居中短暂显示，2秒后消失

**按钮状态：**

- 默认态：正常显示
- 点击态：透明度降低或缩放效果
- 禁用态：灰色背景，不可点击
