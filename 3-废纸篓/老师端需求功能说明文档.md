# 校校读吧小管家 - 教师端需求功能说明文档

## 一、项目概述

### 1.1 产品定位

校校读吧小管家是一个面向中小学的阅读管理平台,旨在帮助教师管理班级阅读活动、监控学生阅读进度、组织阅读测评,提升学生的阅读兴趣和能力

教师端是平台的管理端,为教师提供完整的班级管理、学生监控、计划管理和测评管理功能

### 1.2 目标用户

- 中小学语文教师
- 班主任
- 阅读推广教师
- 学校图书管理员

### 1.3 核心价值

- 简化班级阅读活动的组织和管理
- 实时监控学生阅读进度和成果
- 提供数据化的阅读评估和分析
- 促进师生互动和阅读交流
- 减轻教师的管理负担

### 1.4 技术架构

- 移动端优先设计(375px宽度)
- 响应式HTML5单页应用
- 使用Font Awesome图标库
- 采用莫兰迪色系(低饱和度渐变)
- 支持微信分享功能
- 前端使用原生JavaScript,无框架依赖

## 二、详细功能说明

### 2.1 阅读计划管理

#### 核心功能

- 创建阅读计划：设置计划名称、时间范围、选择书籍
- 计划预览：查看计划详情、书籍列表、参与班级
- 计划发布：将计划发布到指定班级，学生可报名参与
- 进度跟踪：实时监控各班级、各学生的阅读进度
- 完成情况：查看计划整体完成率、优秀学生排名

#### 数据展示

- 计划状态：未开始、进行中、已结束
- 参与统计：参与班级数、参与学生数、完成人数
- 进度可视化：进度条、百分比、颜色分级

### 2.2 班级管理

#### 2.2.1 核心目标

班级管理模块是教师端的核心功能之一，旨在为教师提供完整的班级全生命周期管理能力，从班级创建、学生邀请到日常管理，形成闭环的管理流程

**解决的核心问题**

- 简化班级组织工作：通过邀请链接机制，让学生自主加入班级，无需教师逐个添加
- 提升管理效率：集中管理班级信息、学生名单，避免多平台切换和数据分散
- 促进数据驱动决策：实时查看班级的学生情况，为教学调整提供数据支撑

- 邀请链接生成逻辑：
  - Token生成规则：
    - 前缀："INV"
    - 时间戳：Date.now()
    - 随机字符串：Math.random().toString(36).substring(2, 15)
    - 完整格式：INV{时间戳}{随机字符串}
  - 有效期计算：
    - 创建时间：当前时间
    - 过期时间：创建时间 + 3天（3 * 24 * 60 * 60 * 1000毫秒）
  - 链接格式：
    - {域名}/学生端/join-class.html?classId={班级ID}&token={Token}
    - 示例：https://example.com/学生端/join-class.html?classId=123&token=INV1737360000abc123
  - 数据存储：
    - 保存到localStorage（键名：currentInviteLink）
    - 数据结构：{classId, token, createTime, expireTime, status: 'active'}
    - 实际应用中应保存到后端数据库
- 交互流程：
  - **页面加载流程**：
    1. 检查登录状态（localStorage.getItem('isLoggedIn')）
    2. 未登录则跳转到01-登录.html
    3. 已登录则调用initPage()初始化页面
    4. 如果班级有描述，显示描述区域
    5. 调用renderOngoingPlans()渲染进行中的计划列表
  - **查看计划详情流程**：
    1. 用户点击计划卡片
    2. 跳转到11-计划详情.html?planId={计划ID}
  - **邀请学生流程**：
    1. 用户点击底部"邀请学生加入班级"按钮
    2. 调用showInvitePreview()函数
    3. 从URL参数或classInfo获取classId
    4. 调用generateInviteLink(classId)生成邀请链接和有效期
    5. 将链接和有效期填充到模态框
    6. 显示邀请预览模态框（添加show类）
  - **发送到微信流程**：
    1. 用户点击"发送到微信"按钮
    2. 调用sendToWeChat()函数
    3. 获取邀请链接文本
    4. 尝试使用navigator.clipboard.writeText()复制到剪贴板
    5. 复制成功：
       - 隐藏模态框
       - 显示alert提示："邀请链接已复制到剪贴板！\n\n您可以：\n1. 打开微信\n2. 选择要邀请的学生或家长\n3. 粘贴并发送链接\n\n学生点击链接后确认即可加入班级"
    6. 复制失败或不支持clipboard API：
       - 隐藏模态框
       - 显示alert提示："请手动复制邀请链接发送给学生：\n\n{链接}"
  - **取消邀请流程**：
    1. 用户点击"取消"按钮或点击遮罩层
    2. 调用hideInviteModal()函数
    3. 移除模态框的show类，隐藏模态框
- 数据来源：

**操作路径**

**创建班级流程**

1. 教师在班级列表页点击"创建班级"按钮
2. 跳转到创建班级页面，填写班级信息
3. 点击提交按钮，前端验证表单完整性
4. 验证通过后，调用创建班级接口
5. 后端生成班级ID和邀请链接token，返回班级详情
6. 前端跳转到班级详情页，自动弹出邀请预览模态框
7. 教师可复制邀请链接或分享到微信

**邀请学生流程**

1. 教师在班级详情页或学生管理页点击"邀请学生加入班级"按钮
2. 弹出邀请预览模态框，展示邀请链接和有效期
3. 教师点击"发送到微信"按钮，链接自动复制到剪贴板
4. 教师在微信中粘贴链接并发送给学生或家长
5. 学生点击邀请链接，确认后加入班级
6. 学生加入成功后，班级详情页的学生列表自动更新

**修改班级信息流程**

1. 教师在班级详情页点击右上角设置按钮
2. 跳转到班级设置页，表单预填充当前班级信息
3. 教师修改需要更新的字段
4. 点击保存按钮，调用更新班级接口
5. 更新成功后，返回班级详情页，展示最新信息

**停用/启用班级流程**

1. 教师在班级设置页切换状态开关
2. 前端弹出二次确认对话框，说明状态变更的影响
3. 教师确认后，调用状态切换接口
4. 后端更新班级状态，返回成功响应
5. 前端更新页面状态显示，并提示操作成功

**删除班级流程**

1. 教师在班级设置页点击"删除班级"按钮
2. 前端弹出确认对话框
3. 教师确认后，调用删除班级接口
4. 后端检查是否有报名过的阅读计划
5. 如果有报名记录，返回错误提示"该班级已参与阅读计划，无法删除"
6. 如果无报名记录，执行物理删除，返回成功响应
7. 前端跳转到班级列表页，移除已删除的班级卡片

- 生成时机：教师点击"邀请学生加入班级"按钮时动态生成
- Token格式：'INV' + 时间戳 + 随机字符串
- 链接格式：`${域名}/学生端/join-class.html?classId=${班级ID}&token=${token}`
- 有效期：3天（72小时）
- 存储方式：localStorage存储邀请数据（包括classId、token、创建时间、过期时间、状态）
- 刷新机制：每次点击"邀请学生加入班级"按钮时生成新的邀请链接，旧链接仍在有效期内可用

**莫兰迪色系分配算法**

- 初一：#C9B8A8（暖灰棕）
- 初二：#B8C9C1（雾霾绿）
- 初三：#C1B8C9（薰衣草灰）
- 高一：#C9C1B8（米驼色）

- 分配逻辑：根据班级的年级字段，自动匹配对应的色值，应用于班级卡片的背景渐变

**状态管理逻辑**

- 启用状态（active）：
  - 学生可以通过邀请链接加入班级
  - 学生可以参与该班级的阅读计划
  - 学生可以提交测评答案
  - 班级在列表中正常显示
- 停用状态（inactive）：
  - 学生无法通过邀请链接加入班级（提示"该班级已停用"）
  - 已加入的学生可以查看历史数据，但无法参与新的阅读计划
  - 已加入的学生无法提交新的测评答案
  - 班级在列表中显示灰色标识

**数据刷新机制**

- 班级列表页：进入时加载，支持下拉刷新
- 班级详情页：进入时加载，学生加入后通过轮询（每30秒）自动刷新学生列表
- 邀请预览模态框：打开时动态生成新的邀请链接和有效期

**业务规则**

**邀请链接生成规则**

- Token格式：'INV' + 时间戳 + 随机字符串
- 生成算法：使用Date.now()获取时间戳 + Math.random()生成随机字符串
- 有效期：生成时间 + 72小时（3天）
- 存储方式：localStorage存储邀请数据（包括classId、token、创建时间、过期时间、状态）
- 过期处理：邀请链接过期后自动失效，学生无法使用该链接加入班级
- 生成机制：每次点击"邀请学生加入班级"按钮时动态生成新的邀请链接

**班级删除规则**

- 前置检查：查询class_plans表，检查该班级是否有报名记录
- 有报名记录：返回错误提示，禁止删除，建议教师改为停用状态
- 无报名记录：执行物理删除，同时删除class_students表中的关联记录
- 级联删除：删除班级时，自动删除该班级的所有学生关联记录
- 不可恢复：物理删除后数据无法恢复，需在前端进行二次确认

**班级状态规则**

- 启用状态（active）：
  - 邀请链接有效（未过期时）可以被学生使用
  - 学生可以参与该班级的阅读计划
  - 学生可以提交测评答案
  - 班级在列表中正常显示
- 停用状态（inactive）：
  - 邀请链接无法被使用，学生加入时提示"该班级已停用"
  - 已加入的学生可以查看历史数据（阅读记录、测评成绩）
  - 已加入的学生无法参与新的阅读计划
  - 已加入的学生无法提交新的测评答案
  - 班级在列表中显示灰色标识

**学生加入规则**

- 邀请链接验证：检查邀请token是否存在、是否过期、对应班级是否启用
- 重复检查：检查class_students表，防止学生重复加入同一班级
- 自动加入：验证通过后，直接加入班级，无需教师审核
- 计数更新：加入成功后，classes表的student_count字段自动+1

**学生移除规则**

- 权限检查：只有班级的创建教师可以移除学生
- 数据清理：移除学生时，删除class_students表中的关联记录
- 计数更新：移除成功后，classes表的student_count字段自动-1
- 历史数据：学生的阅读记录和测评成绩保留，不随移除而删除

**权限控制**

**角色定义**

- 教师（Teacher）：可以创建和管理自己的班级
- 学生（Student）：可以通过邀请链接加入班级
- 管理员（Admin）：可以查看和管理所有班级

**权限矩阵**

| 操作             | 教师（自己的班级） | 教师（他人的班级） | 学生 | 管理员     |
| ---------------- | ------------------ | ------------------ | ---- | ---------- |
| 创建班级         | ✓                 | -                  | ✗   | ✓         |
| 查询班级列表     | ✓（仅自己的）     | ✗                 | ✗   | ✓（所有） |
| 查看班级详情     | ✓                 | ✗                 | ✗   | ✓         |
| 更新班级信息     | ✓                 | ✗                 | ✗   | ✓         |
| 删除班级         | ✓                 | ✗                 | ✗   | ✓         |
| 生成邀请链接     | ✓                 | ✗                 | ✗   | ✓         |
| 切换班级状态     | ✓                 | ✗                 | ✗   | ✓         |
| 移除学生         | ✓                 | ✗                 | ✗   | ✓         |
| 通过邀请链接加入 | ✗                 | ✗                 | ✓   | ✗         |

**权限验证流程**

1. 接口调用时，从请求头中提取JWT token
2. 解析token，获取用户ID和角色信息
3. 对于班级相关操作，查询classes表，验证teacher_id是否与当前用户ID一致
4. 如果不一致且用户不是管理员，返回403无权限错误
5. 如果一致或用户是管理员，继续执行业务逻辑

**安全策略**

- 所有接口必须携带有效的JWT token
- Token过期时间：24小时
- 敏感操作（删除班级、移除学生）需要二次验证
- 邀请链接不加密传输，但需要HTTPS协议保障传输安全
- 防止暴力破解：邀请链接验证失败超过5次，锁定10分钟

**存储策略**

**数据库选型**

- 主数据库：MySQL 8.0+（支持事务、外键约束、JSON字段）
- 字符集：utf8mb4（支持emoji和特殊字符）
- 存储引擎：InnoDB（支持事务和行级锁）

**缓存策略**

- 缓存工具：Redis 6.0+

  - 班级列表：key格式 `class:list:teacher:{teacherId}`，过期时间5分钟
  - 班级详情：key格式 `class:detail:{classId}`，过期时间10分钟
  - 邀请token映射：key格式 `invite:token:{inviteToken}`，过期时间与邀请链接一致（3天）
- 缓存更新策略：
  - 创建/更新/删除班级时，删除对应的缓存key
  - 学生加入/移除时，删除班级详情缓存
  - 采用Cache-Aside模式：先查缓存，未命中则查数据库并写入缓存

**异常处理逻辑**

**异常分类**

- 系统异常（500）：数据库连接失败、Redis连接失败、第三方服务异常
- 业务异常（400）：参数验证失败、业务规则不满足
- 权限异常（403）：无权限操作他人的班级
- 资源异常（404）：班级不存在、学生不存在

**错误码设计**

| 错误码 | HTTP状态码 | 说明           | 示例                           |
| ------ | ---------- | -------------- | ------------------------------ |
| 200    | 200        | 成功           | 操作成功                       |
| 400    | 400        | 参数错误       | 班级名称不能为空               |
| 401    | 401        | 未登录         | 请先登录                       |
| 403    | 403        | 无权限         | 您没有权限操作该班级           |

| 409    | 409        | 资源冲突       | 邀请token已存在                |
| 422    | 422        | 业务规则不满足 | 该班级已参与阅读计划，无法删除 |
| 500    | 500        | 服务器错误     | 服务器内部错误，请稍后重试     |
| 503    | 503        | 服务不可用     | 服务暂时不可用，请稍后重试     |

**异常处理流程**

1. 捕获异常：使用try-catch捕获所有异常
2. 记录日志：将异常信息记录到日志文件，包括请求参数、用户信息、堆栈信息
3. 分类处理：
   - 系统异常：返回500错误，隐藏具体错误信息，避免泄露系统细节
   - 业务异常：返回对应的错误码和友好的错误提示
   - 权限异常：返回403错误，提示无权限
   - 资源异常：返回404错误，提示资源不存在
4. 返回响应：统一的错误响应格式

```json
{
  "code": 400,
  "message": "班级名称不能为空",
  "data": null,
  "timestamp": "2026-01-20T10:30:00",
  "path": "/api/classes"
}
```

**重试机制**

- 数据库操作失败：自动重试3次，每次间隔1秒
- Redis操作失败：降级处理，直接查询数据库
- 第三方服务调用失败：重试2次，超时时间5秒

**熔断机制**

- 当数据库连接失败率超过50%时，触发熔断，返回503错误
- 熔断时间：5分钟
- 熔断恢复：5分钟后自动尝试恢复，如果成功则解除熔断

**监控告警**

- 监控指标：接口响应时间、错误率、数据库连接数、Redis连接数
- 告警阈值：
  - 接口响应时间P95 > 1秒
  - 错误率 > 5%
  - 数据库连接数 > 80%
  - Redis连接数 > 80%
- 告警方式：短信、邮件、钉钉/企业微信机器人

```javascript
{
    id: Number,              // 计划ID

    startDate: String,       // 开始日期 (YYYY-MM-DD)
    endDate: String,         // 结束日期 (YYYY-MM-DD)
    status: String,          // 状态: 'ongoing' | 'ended' | 'pending'
    statusText: String,      // 状态文本: '进行中' | '已结束' | '未开始'
    books: Array<Book>       // 书籍列表

```

```javascript
{
    id: Number,              // 书籍ID
    name: String,            // 书籍名称
    examCount: Number,       // 测评题数量（固定20题）
    submitted: Number,       // 已提交人数
    total: Number,           // 总人数
    submitRate: Number,      // 提交率（百分比）
    avgScore: Number         // 平均分
}
```

```javascript
{
    id: Number,              // 班级ID

    semester: String,        // 学期: '上学期' | '下学期'
    studentCount: Number,    // 学生人数
    planCount: Number,       // 计划数量
    teacher: String,         // 教师姓名
    description: String,     // 班级描述
    status: String,          // 状态: 'active' | 'inactive'
```

### 3.2 状态枚举

#### 计划状态

- `pending`: 未开始
- `ongoing`: 进行中
- `ended`: 已结束

#### 班级状态

- `active`: 启用
- `inactive`: 停用

#### 提交状态

- `submitted`: 已提交
- `pending`: 未提交
