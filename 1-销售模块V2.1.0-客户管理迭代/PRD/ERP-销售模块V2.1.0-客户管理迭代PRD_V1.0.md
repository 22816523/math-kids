## 1. 背景与目标

### 1.1 项目背景

本次迭代涉及 ERP 系统与作者资源库系统两个部分。

ERP 系统当前联系人数据以从表为主，并与业务经理关联，缺少联系人主表来承载统一联系人数据；单位（学校等）和联系人相关数据也存在字段补充需求。

作者资源库系统在单位和作者信息维护过程中，需要与 ERP 建立明确的数据获取与同步关系，以支撑所在单位选择、单位新增、作者复审和作者信息修改等业务场景。

### 1.2 产品目标

| 目标类型 | 具体目标 |
| -------- | -------- |
| 业务目标 | 建立 ERP 联系人主表及主从关系管理基础 |
| 业务目标 | 补齐 ERP 单位（学校等）和联系人本期所需数据字段 |
| 用户目标 | 满足联系人查看和导出场景中的敏感信息处理要求 |
| 用户目标 | 支持作者资源库与 ERP 之间的关键数据联动 |
| 技术目标 | 明确 ERP 与作者资源库之间本期涉及的数据来源和同步口径 |

### 1.3 项目范围

- ERP 联系人增加主表功能，现有联系人从表继续保留并与业务经理关联。
- ERP 联系人主表与从表新增字段：身份证号码、学科。
- ERP 系统基础类型新增【学科】类型，学科下拉与导入校验均取该类型维护值。
- ERP 联系人主表新增字段：联系人来源系统。
- ERP 联系人主表新增导入功能。
- ERP 联系人列表手机号脱敏处理，详情页可以查看完整手机号。
- ERP 导出联系人时过滤联系人电话，具体导出信息待定。
- ERP 联系人复制时新增从库数据，并记录对应主库数据。
- ERP 联系人审核时，从库新增数据同步给主库；若主库已存在，则只记录对应关系。
- ERP 单位（学校等）新增学校性质字段，支持公立、民办。
- 作者资源库作者管理中的所在单位改为下拉选择，支持输入内容模糊匹配，数据从 ERP 单位（学校等）获取。
- 作者资源库作者管理列表增加【新增单位】按钮，新增单位后同步给 ERP；同步到 ERP 的单位状态默认已开通，类型为普通。
- 作者复审通过后，将作者信息同步到 ERP 联系人主表。
- 作者信息修改场景下，将姓名、手机号、身份证号、所在单位、学科的修改同步到 ERP。

### F001 联系人管理

**功能说明**

本期新增 `ERP系统-销售-客户管理-联系人管理` 菜单。该页用于维护联系人主表，并联动查看对应从表/业务经理关联信息；页面提供联系人查询、导入、新增、开通业务经理、编辑、查看和导出，是联系人主数据的统一入口。

**业务规则**

1. 主表列表数据取自 ERP 联系人主表，从表关联明细取自联系人与业务经理关系数据。
2. 联系人主表由现有从表数据初始化生成，姓名、单位、手机号三项完全一致视为同一联系人。
3. 联系人主表ID由系统生成，展示格式为 `LX00000001` 这类编码。
4. 主表状态仅包含启用、停用；从表保留待审核、启用、停用、退回等状态。
5. 手机号、身份证号全部脱敏展示；电话可为空，身份证号可为空。手机号示例：`1598****9070`；身份证号示例：`522326********12`。
6. 学科字段取 `ERP-系统-基础设置-基础类型` 中类型【学科】维护的值。
7. 联系人新增、编辑、导入、取消关联等操作均以当前选中的联系人主表或关联记录为处理对象；开通业务经理支持勾选多个联系人主表后批量开通。

**界面要素**

1. 菜单入口：`ERP系统-销售-客户管理-联系人管理`。

**查询区**

### F002 新增联系人

**功能说明**

新增联系人用于在联系人管理页创建联系人主表记录，并同步维护基础信息、地址信息和来源系统字段。

**业务规则**

1. 入口位于联系人管理页工具栏【新增】，不依赖当前列表选中记录。
2. 新增保存后直接写入联系人主表，不走审核流程，主表状态默认为启用。
3. 联系人来源系统由系统按来源规则自动判定并写入 ERP，页面中不提供手工维护入口。
4. 查重规则为姓名、单位、手机号三项同时一致判定重复；查重命中时不重复生成主档，并提示用户改用已有主档处理业务经理关系。
5. 学科字段取自 `ERP-系统-基础设置-基础类型` 中类型【学科】维护的值。
6. 省份、城市、区县按所选单位自动带出，到站、街道、地址可在此基础上调整。
7. 新增表单与导入模板保持同一字段口径。

**界面要素**

| 元素 | 类型 | 说明 |
| ---- | ---- | ---- |
| 单位名称 | 下拉选择 | 从 ERP 单位（学校等）中选择，必填；切换单位后自动带出省份、城市、区县。 |
| 联系人 | 输入框 | 必填，用于联系人姓名。 |
| 手机号 | 输入框 | 必填，用于联系人手机号码，并参与查重。 |
| 电话 | 输入框 | 可选，用于固定电话或办公电话。 |

**异常处理**

1. 必填项缺失时提示补全后再提交。
2. 单位名称无法匹配或学科取值不合法时，阻止提交并提示原因。
3. 查重命中但未处理时，不直接重复生成主档。

### F003 编辑联系人

**功能说明**

编辑联系人用于修改联系人主表已存在的数据。

**业务规则**

1. 编辑入口基于当前选中的联系人主表记录打开。
2. 可修改单位名称、联系人、手机号、电话、学科、身份证号、性别、职务、部门、年级、到站、街道、地址等基础字段。
3. 联系人来源系统由系统维护，不作为人工编辑字段。
4. 编辑保存后直接更新联系人主表，不影响已存在的业务经理关系。
5. 保存前按与新增一致的姓名、单位、手机号规则进行重复校验；命中重复时阻止提交并提示原因。

**界面要素**

| 元素 | 类型 | 说明 |

| 弹窗标题 | 文本 | 显示“编辑联系人”。 |
| 字段布局 | 表单 | 与新增联系人共用同一字段布局，当前记录值全部回填。 |
| 单位名称 | 下拉选择 | 回填当前单位，可修改。 |
| 联系人 / 手机号 / 电话 / 学科 / 身份证号 / 性别 / 职务 / 部门 / 年级 | 表单项 | 回填当前值并允许修改。 |
| 省份 / 城市 / 区县 | 文本 | 由当前单位带出，只读展示。 |
| 到站 / 街道 / 地址 | 输入框 | 回填当前值，可修改。 |
| 保存 | 按钮 | 保存修改后的联系人主表信息。 |
| 取消 | 按钮 | 关闭弹窗，不保存。 |

**异常处理**

1. 未选择联系人时，点击编辑应提示先选择记录。
2. 保存失败时保留当前输入内容。
3. 重复校验未通过时，不提交并提示重复原因。

查看联系人用于只读查看联系人主表详情。

### F005 联系人导入

**功能说明**

联系人导入用于批量导入联系人主表数据，并完成模板校验、重复校验和结果反馈。

**业务规则**

1. 入口位于联系人管理页工具栏【导入】。
2. 仅支持 `.xlsx` 文件，单次最多 5000 条。
3. 导入模板与新增联系人保持同一字段口径，模板共 17 列，必填列以模板定义为准。
4. 新增和导入的联系人主表数据直接保存，不经过审核。
5. 姓名、单位、手机号三项同时一致判定重复；重复行不重复生成主档。
6. 学科为本次主从表新增必填字段，取值必须命中 `ERP-系统-基础设置-基础类型` 中类型【学科】。
7. 身份证号非必填，联系人来源系统由系统自动写入 ERP，模板中不维护该列。
8. 业务经理列可选，未填写时默认当前导入人，多业务经理用英文逗号分隔。
9. 校验通过后，主档不存在则创建联系人主表并开通所选业务经理关系；主档已存在且未开通业务经理时，仅补充联系人与业务经理关系。
10. 单位名称无法匹配或同业务经理名下重复时，阻断该行导入并返回问题原因。
11. 导入结果返回成功、重复、失败等明细，便于定位问题行。

**界面要素**

| 元素 | 类型 | 说明 |
| ---- | ---- | ---- |
| 导入流程 | 步骤说明 | 依次为下载模板、按模板维护数据、上传文件、执行校验、创建主档或补充业务经理关系。 |
| 下载模板 | 按钮 | 下载与当前导入规则一致的联系人导入模板。 |
| 文件上传 | 上传控件 | 支持拖拽或选择文件，仅支持 `.xlsx`，单次最多 5000 条。 |
| 导入规则说明 | 文本 | 展示必填列、查重列、默认来源和处理口径。 |
| 导入结果反馈 | 表格 | 展示成功、重复、失败及其原因，支持定位问题行。 |

**异常处理**

1. 文件格式不符合要求时，提示无法导入。
2. 数据校验不通过时，阻止提交并显示问题行。
3. 单位名称无法匹配或业务经理重复时，返回行级失败原因。

### F006 联系人导出

**功能说明**

联系人导出用于将联系人管理页当前筛选结果导出为文件，并按规则处理电话字段和来源口径。

**业务规则**

1. 导出范围以当前筛选条件命中的全部联系人为准，不限当前页。
2. 导出文件中的电话字段不输出明文，按导出规则统一过滤或脱敏。
3. 对“无来源新增”和“有来源新增”记录按联系人来源系统是否存在分别处理导出值口径。
4. 导出结果保持与列表查询条件一致，便于和当前查询范围对齐。

**界面要素**

| 元素 | 类型 | 说明 |
| ---- | ---- | ---- |
| 导出 | 按钮 | 导出当前筛选结果。 |
| 导出结果提示 | 提示 | 反馈导出成功、失败或无数据。 |

**异常处理**

1. 无可导出数据时，应提示当前条件下无数据。
2. 导出失败时，应提示失败原因。

### F007 开通业务经理

**功能说明**

开通业务经理用于为已勾选的一个或多个联系人主表批量开通业务经理关系。

**业务规则**

1. 入口位于联系人管理页工具栏【开通业务经理】，仅对已勾选的联系人主表生效。
2. 开通前必须先选中至少一条联系人主表记录；未选中时不进入业务经理选择流程。
3. 弹窗加载业务经理目录数据，支持按姓名或工号搜索、分页和多选。
4. 已选业务经理可单个删除或一键清空。
5. 确认后为当前勾选的联系人主表批量开通所选业务经理关系。
6. 业务经理关系的创建对象是联系人主表，不是列表中的从表关联记录。

**界面要素**

| 元素 | 类型 | 说明 |
| ---- | ---- | ---- |
| 业务经理搜索框 | 输入框 | 支持按业务经理姓名或工号模糊搜索。 |
| 业务经理列表 | 列表/复选框 | 展示姓名和工号，支持单页多选和全选。 |
| 分页 | 分页控件 | 切换业务经理候选页。 |
| 已选业务经理汇总区 | 文本 | 展示已选人数和当前选择状态。 |
| 已选业务经理列表 | 标签列表 | 展示已选业务经理，支持单个删除。 |
| 清空已选 | 按钮 | 清空当前已选业务经理。 |
| 取消 / 确认开通 | 按钮 | 取消关闭弹窗，确认后执行批量开通。 |

**异常处理**

1. 未选择业务经理时，不允许确认开通。
2. 搜索无结果时，提示未找到匹配人员。
3. 未勾选联系人主表时，入口不执行并提示先选择记录。

### F008 联系人复制

**功能说明**

联系人复制为存量功能，本期仅补齐主表引入后的数据口径，不重复描述原有复制动作。

**业务规则**

1. 复制结果必须保留联系人主表关联信息，用于后续审核和主表追溯。
2. 复制结果按主表口径补齐并落库身份证号码、学科、来源系统等新增字段。
3. 除主表新增字段外，其余复制规则沿用存量逻辑。

### F009 联系人审核

**功能说明**

联系人审核为存量功能，本期仅补齐主表引入后的审核同步口径，不重复描述原有审核动作。

**业务规则**

1. 审核对象仍为联系人从表待审核记录，主表联系人不走审核流程。
2. 审核通过后，主库不存在则同步新增到主库；主库已存在则只记录对应关系。
3. 审核结果需同步主表口径字段，包含身份证号、学科、来源系统等新增字段。
4. 审核列表和审核详情中的手机号、身份证号码按脱敏规则展示。
5. 审核页的修改/详情弹窗需带出身份证号码、学科、来源系统，其中来源系统只读不可修改，修改场景仅允许调整身份证号码和学科。

### F010 联系人合并

**功能说明**

联系人合并为存量功能，本期仅补齐合并页联系人主从表的新字段展示口径，不重复描述原有合并动作。

**业务规则**

1. ERP系统-销售-客户管理-联系人合并菜单的上方联系人主表和下方联系人从表，均需补齐身份证号码、学科、来源系统字段展示。
2. 联系人合并页点击联系人编码查看详情时，详情弹窗需同步展示身份证号码、学科、来源系统字段。

### F011 单位管理

**功能说明**

ERP系统-销售-客户管理-单位管理、单位合并、单位审核三个菜单，本期仅补齐单位新增字段学校性质及其在相关页面中的展示口径，不重复描述原有单位维护动作。

**业务规则**

1. ERP系统-销售-客户管理-单位管理菜单的单位主数据新增学校性质字段，取值仅包含公立、民办。
2. ERP系统-销售-客户管理-单位管理菜单下的新增单位、修改单位、查看单位场景都需要带出学校性质；其中新增和修改场景可维护，查看场景只读展示。
3. ERP系统-销售-客户管理-单位管理菜单下的联系人从表需同步补齐身份证号码、学科字段，手机号按全量脱敏规则展示，不输出明文。
4. ERP系统-销售-客户管理-单位合并菜单的单位列表需要带出学校性质字段。
5. ERP系统-销售-客户管理-单位审核菜单的单位列表需要带出学校性质字段。
