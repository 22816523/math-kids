## 1. 背景与目标

### 1.1 项目背景

本次迭代涉及 ERP 系统与作者资源库系统两个部分。

ERP 系统当前联系人数据以从表为主，并与业务经理关联，缺少联系人主表来承载统一联系人数据；单位（学校等）和联系人相关数据也存在字段补充需求。

作者资源库系统在单位和作者信息维护过程中，需要与 ERP 建立明确的数据获取与同步关系，以支撑所在单位选择、单位新增、作者复审和作者信息修改等业务场景。

### 1.2 产品目标

| 目标类型 | 具体目标 |
| -------- | -------- |
| 业务目标 | 建立 ERP 联系人主表及主从关系管理基础 |
| 业务目标 | 补齐 ERP 单位（学校等）和联系人本期所需数据字段 |
| 用户目标 | 满足联系人查看和导出场景中的敏感信息处理要求 |
| 用户目标 | 支持作者资源库与 ERP 之间的关键数据联动 |
| 技术目标 | 明确 ERP 与作者资源库之间本期涉及的数据来源和同步口径 |

### 1.3 项目范围

- ERP 联系人增加主表功能，现有联系人从表继续保留并与业务经理关联。
- ERP 联系人主表与从表新增字段：身份证号码、学科。
- ERP 联系人主表新增字段：联系人来源系统。
- ERP 联系人主表新增导入功能。
- ERP 联系人列表手机号脱敏处理，详情页可以查看完整手机号。
- ERP 导出联系人时过滤联系人电话，具体导出信息待定。
- ERP 联系人复制时新增从库数据，并记录对应主库数据。
- ERP 联系人审核时，从库新增数据同步给主库；若主库已存在，则只记录对应关系。
- ERP 单位（学校等）新增学校性质字段，支持公立、民办。
- 作者资源库作者管理中的所在单位改为下拉选择，支持输入内容模糊匹配，数据从 ERP 单位（学校等）获取。
- 作者资源库作者管理列表增加【新增单位】按钮，新增单位后同步给 ERP；同步到 ERP 的单位状态默认已开通，类型为普通。
- 作者复审通过后，将作者信息同步到 ERP 联系人主表。
- 作者信息修改场景下，将姓名、手机号、身份证号、所在单位、学科的修改同步到 ERP。

## 4. 功能详情

### F001 联系人管理

**功能说明**

入口为 `ERP系统-销售-客户管理-联系人管理` 菜单。该页用于维护联系人主表，并联动查看对应从表/业务经理关联信息；页面提供联系人查询、导入、新增、开通业务经理、编辑、查看和导出，是联系人主数据的统一入口。

**业务规则**

1. 主表状态仅包含启用、停用；从表保留待审核、启用、停用、退回等状态。
2. 联系人状态、联系人主表ID、联系人、手机号、电话、学科、单位名称、省份、城市、区县、性别共同构成查询条件。

    2.1 查询匹配方式：联系人主表ID精确匹配，其余字段支持模糊匹配；多个查询条件同时填写时按且关系筛选；查询条件为空时返回全部主表数据。

    2.2 下拉项规则：
        - 联系人状态为固定枚举下拉，取值为启用、停用。
        - 学科为固定枚举下拉，取值为数学、语文、英语、物理。
        - 性别为固定枚举下拉，取值为男、女、未知。
        - 省份/城市/区县为级联下拉，选项取自 ERP 单位（学校等）已同步地址数据并去重生成。
    2.3 数据来源：主表列表数据取自 ERP 联系人主表；从表/业务经理关联信息取自联系人与业务经理关系数据；省份/城市/区县下拉取自 ERP 单位地址同步数据。
    2.4 分页与联动：列表默认分页展示，每页 50 条；查询和重置均回到第一页；切换分页时保留当前筛选条件；选中某条主表联系人后，下方从表/业务经理关联信息随之刷新。
3. 联系人主表初始化时，以现有从表数据为基础生成主表记录；初始化去重规则为姓名、单位、手机号三项完全一致视为同一联系人。
4. 联系人主表ID由系统生成，展示格式为 `LX00000001` 这类编码。
5. 手机号在列表中按权限脱敏展示，查看场景可查看完整手机号。
6. 电话、身份证号可为空；身份证号在界面中按权限脱敏展示。
7. 地址字段由省份、城市、区县、到站、街道、地址组成，体现单位归属信息。
8. 编辑、查看、开通业务经理均基于当前选中的主表联系人执行。

**界面要素**

1. 菜单入口：`ERP系统-销售-客户管理-联系人管理`。
2. 查询区：联系人状态、联系人主表ID、联系人、手机号、电话、学科、单位名称、省份、城市、区县、性别、查询、重置。
3. 操作区：导入、新增、开通业务经理、编辑、查看、导出。
4. 主表列表区：联系人状态、联系人主表ID、联系人、手机号、电话、身份证号、学科、单位名称、省份、城市、区县、到站、街道、地址、创建人、创建时间、性别、职务、部门、修改人、修改时间、审核人、审核时间；支持分页展示，每页 50 条。
5. 从表关联区：当前选中主表联系人对应的业务经理关联明细及取消关联操作。

**验收标准**

1. 能按查询条件筛选并重置列表。
2. 联系人主表ID以系统编码格式展示，并可按完整编码定位记录。
3. 手机号在列表中默认脱敏，查看场景可见完整值。
4. 选中主表联系人后，下方关联信息会刷新为该联系人对应数据。
5. 编辑、查看、开通业务经理入口在未选中联系人时应提示先选择记录。

**异常处理**

1. 查询条件为空时，返回全部数据。
2. 无匹配数据时，列表显示为空结果。
3. 查询条件组合不合法或系统异常时，提示查询失败并保留当前筛选项。

### F002 新增联系人

**功能说明**

新增联系人用于创建联系人主表数据，并同步维护联系人相关基础字段与地址信息。

**业务规则**

1. 新增联系人保存后直接生成主表数据，不走审核流程。
2. 查重规则为姓名、单位、手机号三项同时一致判定重复。
3. 新增时联系人状态默认为启用。
4. 主表维护联系人来源系统字段。
5. 主从表均需维护身份证号、学科等关联字段。

**界面要素**

1. 主档信息：单位名称、联系人、手机号、电话、学科、身份证号、性别、职务、部门、年级。
2. 单位地址：省份、城市、区县、到站、街道、地址。
3. 操作区：查重、保存、取消。

**验收标准**

1. 必填字段校验不通过时，不能保存。
2. 查重命中时给出重复提示，且能继续做业务经理开通处理。
3. 保存后联系人主表可在联系人管理页查询到。

**异常处理**

1. 必填项缺失时提示补全后再提交。
2. 查重命中但未处理时，不直接重复生成主档。

### F003 编辑联系人

**功能说明**

编辑联系人用于修改联系人主表已存在的数据。

**业务规则**

1. 编辑入口基于当前选中的联系人主表记录打开。
2. 可修改联系人基础信息、单位信息和地址信息。
3. 编辑保存后同步更新主表数据。
4. 编辑模式下业务经理关系不在本功能内维护。

**界面要素**

1. 编辑弹窗与新增弹窗共用字段布局。
2. 标题显示为编辑联系人。
3. 操作区为保存、取消。

**验收标准**

1. 打开编辑时可回填当前联系人信息。
2. 保存后列表中的对应记录更新成功。
3. 编辑过程中不影响其他联系人记录。

**异常处理**

1. 未选择联系人时，点击编辑应提示先选择记录。
2. 保存失败时保留当前输入内容。

### F004 查看联系人

**功能说明**

查看联系人用于只读查看联系人主表详情。

**业务规则**

1. 查看入口基于当前选中的联系人主表记录打开。
2. 查看模式下所有字段只读，不允许修改。
3. 查看模式下隐藏查重和保存，仅保留关闭。

**界面要素**

1. 与新增、编辑共用同一弹窗结构。
2. 标题显示为查看联系人。
3. 字段按只读状态展示。

**验收标准**

1. 可完整查看联系人主表信息。
2. 关闭后不产生任何数据变更。

**异常处理**

1. 未选择联系人时，点击查看应提示先选择记录。

### F005 联系人导入

**功能说明**

联系人导入用于批量导入联系人主表数据，并完成模板校验、重复校验和结果反馈。

**业务规则**

1. 新增和导入的联系人主表数据直接保存，不经过审核。
2. 导入时按姓名、单位、手机号进行查重。
3. 导入模板字段需与新增联系人必填字段保持一致。
4. 导入结果需返回成功、重复、失败等处理结果。

**界面要素**

1. 模板下载。
2. 文件上传。
3. 数据校验结果展示。
4. 导入结果反馈表。

**验收标准**

1. 模板可下载，且模板字段与导入规则一致。
2. 文件校验通过后可进入导入处理。
3. 导入结束后可查看成功与失败明细。

**异常处理**

1. 文件格式不符合要求时，提示无法导入。
2. 数据校验不通过时，阻止提交并显示问题行。

### F006 联系人导出

**功能说明**

联系人导出用于将联系人管理页当前筛选结果导出为文件。

**业务规则**

1. 导出范围以当前筛选条件为准。
2. 导出时需按规则处理联系人电话字段。
3. 对“无来源新增”和“有来源新增”场景按不同口径导出。

**界面要素**

1. 导出按钮。
2. 导出结果提示。

**验收标准**

1. 能导出当前筛选范围内的数据。
2. 电话字段按规则脱敏或过滤后输出。

**异常处理**

1. 无可导出数据时，应提示当前条件下无数据。
2. 导出失败时，应提示失败原因。

### F007 开通业务经理

**功能说明**

开通业务经理用于为已存在的联系人主表开通一个或多个业务经理关系。

**业务规则**

1. 入口位于联系人管理页工具栏。
2. 仅对已存在的联系人主表执行开通。
3. 支持搜索、分页、勾选多个业务经理。
4. 已选业务经理可单个删除或一键清空。
5. 确认后完成业务经理关系开通。

**界面要素**

1. 业务经理搜索框。
2. 业务经理列表与分页。
3. 已选业务经理汇总区。
4. 确认开通、取消。

**验收标准**

1. 能按姓名或工号筛选业务经理。
2. 可勾选多个业务经理并保留已选结果。
3. 可删除单个已选业务经理。
4. 确认后完成开通并关闭弹窗。

**异常处理**

1. 未选择业务经理时，不允许确认开通。
2. 搜索无结果时，提示未找到匹配人员。
